## 1. Earthworm 개요

### 1.1 Earthworm이란?

**Earthworm**은 미국 지질조사국(USGS)이 개발한 오픈소스 지진 데이터 수집·처리·배포 플랫폼입니다.  
1990년대 초반에 개발이 시작된 이후, 전 세계 수백여 지진 관측 네트워크에서 운용되고 있으며 실시간 지진 관측소 데이터의 표준 처리 시스템으로 자리잡았습니다.

Earthworm의 핵심 철학은 **모듈성(Modularity)** 입니다. 시스템은 독립적으로 동작하는 다수의 모듈(프로세스)로 구성되며, 각 모듈은 **공유 메모리 링(Transport Ring)** 을 통해 데이터를 교환합니다.

#### ✅ 주요 특징

| 특징 | 설명 |
|------|------|
| 통합 프레임워크 | 수집 → 처리 → 분석 → 배포를 하나의 시스템에서 관리 |
| 모듈 기반 아키텍처 | 기능 단위로 프로세스를 자유롭게 추가/제거 |
| 고속 IPC | Transport Ring(공유 메모리)을 이용한 프로세스 간 통신 |
| 다양한 프로토콜 | SeedLink, LISS, IRISC 등 지원 |
| 멀티 플랫폼 | Linux, Windows, macOS 운용 가능 |
| 자동 분석 | 자동 이벤트 감지, 진앙 결정, ShakeMap 연동 |

---

### 1.2 시스템 아키텍처 개요

Earthworm의 데이터 흐름은 크게 세 단계로 구성됩니다.

```
┌─────────────────────────────────────────────────────────────┐
│                    Earthworm 데이터 흐름                     │
│                                                             │
│  [지진계/외부네트워크]                                        │
│         │                                                   │
│         ▼                                                   │
│  ┌─────────────┐    ┌──────────────┐    ┌───────────────┐  │
│  │  수집 모듈   │───▶│  WAVE_RING   │───▶│   처리 모듈   │  │
│  │ slink2ew    │    │ (공유메모리) │    │ pick_ew       │  │
│  │ import_gen  │    │  Transport   │    │ binder_ew     │  │
│  │ waveserverV │    │    Ring      │    │ hypo2000      │  │
│  └─────────────┘    └──────────────┘    └───────────────┘  │
│                            │                               │
│                            ▼                               │
│                    ┌───────────────┐                       │
│                    │   배포 모듈   │                       │
│                    │ export_gen    │                       │
│                    │ ew2slink      │                       │
│                    │ ringserver    │                       │
│                    └───────────────┘                       │
└─────────────────────────────────────────────────────────────┘
```

| 단계 | 구성 요소 | 설명 |
|------|-----------|------|
| **수집 (Ingest)** | waveserverV, import_generic, slink2ew | 지진계 또는 외부 네트워크로부터 파형 수신 |
| **처리 (Process)** | pick_ew, binder_ew, hypo2000 | 자동 P파 픽킹, 진앙 결정, 규모 산출 |
| **배포 (Export)** | export_generic, ew2slink, ringserver | 처리 결과를 외부 시스템 또는 사용자에게 전송 |

---

### 1.3 핵심 개념: Transport Ring

Transport Ring은 **공유 메모리(Shared Memory) 방식의 순환 버퍼(Circular Buffer)** 입니다.  
모든 Earthworm 모듈은 이 링에 연결(attach)하여 메시지를 읽거나 씁니다.

| 구성 요소 | 설명 |
|-----------|------|
| **Ring Key** | 링을 식별하는 고유 정수 키 (예: `WAVE_RING = 1000`) |
| **Ring Size** | 링의 총 크기(바이트), 설정 파일에서 지정 |
| **Message** | `TYPE + MODULE + INST + Sequence` 헤더 + Payload |
| **Logo** | TYPE, MODULE, INSTALLATION 3종의 식별자 조합 |
| **Sequence #** | 단조증가 메시지 번호 — 유실 감지에 활용 |

> ⚠️ **주의**: 링이 가득 차면 가장 오래된 메시지부터 덮어씁니다. 느린 소비자 모듈은 메시지를 유실할 수 있으므로, 대용량 환경에서는 Ring 크기를 충분히 크게 설정해야 합니다.

## 3. Transport Ring 심화

### 3.1 링(Ring) 크기 설정 — `startstop.d`

```ini
# startstop.d  –  Ring 정의 예시 (대용량 구성)

RingSize  WAVE_RING     524288000   # 500 MB  : 파형 데이터 메인 버퍼
RingSize  PICK_RING      32768000   #  32 MB  : 픽 결과 저장
RingSize  HYPO_RING      16384000   #  16 MB  : 진앙 결정 결과
RingSize  STATUS_RING     4096000   #   4 MB  : 모듈 상태 정보
RingSize  REPORT_RING     8192000   #   8 MB  : 알람 및 리포트

# Ring Key는 earthworm.d 에서 정의
RingKey   WAVE_RING     1000
RingKey   PICK_RING     1005
RingKey   HYPO_RING     1010
```

---

### 3.2 주요 메시지 타입(MSG TYPE)

| 메시지 타입 | Logo # | 용도 | 주요 생산 모듈 |
|-------------|--------|------|----------------|
| `TYPE_TRACEBUF2` | 20 | **파형 데이터 (가장 핵심)** | waveserverV, import_generic |
| `TYPE_PICK_SCNL` | 10 | P파 도달 시각 및 특성 | pick_ew, carlstatrig |
| `TYPE_CODA_SCNL` | 11 | S파 코다 정보 | pick_ew |
| `TYPE_HYPOINVERSE` | — | 진앙 위치 결정 결과 | hypo2000, binder_ew |
| `TYPE_HYP2000ARC` | 12 | HYP2000 아카이브 포맷 | hypo2000 |
| `TYPE_STRONGMOTION` | 101 | 강진동 가속도 데이터 | sm_trig |
| `TYPE_HEARTBEAT` | 6 | 모듈 생존 확인 신호 | 모든 모듈 |
| `TYPE_ERROR` | 9 | 오류 보고 | 모든 모듈 |
| `TYPE_MSEED` | 60 | mini-SEED 레코드 | ew2slink |

### 3.3 TRACEBUF2 헤더 구조

```c
// TRACEBUF2 헤더 구조 (C 구조체)
typedef struct {
    double  starttime;      // 샘플 시작 시각 (epoch seconds)
    double  endtime;        // 샘플 종료 시각
    double  samprate;       // 샘플링 레이트 (Hz)
    char    sta[7];         // 관측소 코드
    char    net[9];         // 네트워크 코드
    char    chan[9];        // 채널 코드
    char    loc[3];         // 위치 코드
    char    version[2];     // 버전 ("20")
    char    datatype[3];    // 데이터 타입 ("i4", "s4", "f4" 등)
    short   quality;        // 데이터 품질 플래그
    short   pad;            // 패딩
    int     nsamp;          // 샘플 수
} TraceHeader;
// 이후 int32_t 또는 float 배열로 샘플 데이터가 이어짐
```

## 4. 대용량 데이터 수신 (Import)

### 4.1 `import_generic` — TCP 파형 수신

외부 시스템으로부터 TCP/IP 소켓을 통해 Earthworm 메시지를 수신하는 표준 모듈입니다.

```ini
# import_generic.d  –  주요 파라미터

MyModuleId       MOD_IMPORT_GEN
RingName         WAVE_RING          # 수신 데이터를 쓸 링
HeartBeatInt     30                 # 하트비트 전송 간격(초)

# 수신 소켓 설정
InSocketNum      16005              # 수신 포트 번호
MaxMsgSize       65536              # 최대 메시지 크기 (바이트)

# 클라이언트 모드 (원격 서버에 연결)
ServerSide       0                  # 0=클라이언트, 1=서버
RemoteAddress    192.168.1.100      # 원격 호스트 IP
RemotePort       16005              # 원격 포트

# 수신할 메시지 타입 필터 (화이트리스트)
GetLogo  TYPE_TRACEBUF2  MOD_WILDCARD  INST_WILDCARD
GetLogo  TYPE_PICK_SCNL  MOD_WILDCARD  INST_WILDCARD

# 재연결 설정 (대용량 환경 핵심)
ReconSec         5                  # 연결 끊김 시 재연결 간격(초)
SocketTimeout    30                 # 소켓 타임아웃(초)
```

---

### 4.2 `slink2ew` — SeedLink 대용량 파형 수신

**SeedLink**는 FDSN 표준 실시간 파형 전송 프로토콜입니다. `slink2ew` 모듈로 SeedLink 서버(SeisComp, ringserver 등)로부터 데이터를 수신합니다.

```ini
# slink2ew.d  –  SeedLink 수신 설정

MyModuleId       MOD_SLINK2EW
RingName         WAVE_RING
HeartBeatInt     30

# SeedLink 서버 정보
SLhost           seedlink.kma.go.kr:18000   # 기상청 SeedLink 예시

# 방법 1: 전체 네트워크 수신 (대용량 수신 시 권장)
AllNetworks      1

# 방법 2: 특정 스테이션/채널 선택
# Stream           KS_SEO_BHZ_00
# Stream           KS_SEO_BHN_00

SeedRecordSize   512     # 1이면 mini-SEED를 TYPE_TRACEBUF2로 변환
StateFile        slink2ew_state.txt   # 재시작 시 이어받기
TimeTol          0.5     # 시간 허용 오차(초)
MaxGap           86400   # 최대 데이터 갭 허용(초)
```

> 💡 `StateFile`을 설정해두면 프로세스 재시작 후 마지막으로 수신한 시퀀스 번호부터 이어 받아 **데이터 유실을 방지**합니다.

### 4.3 `waveserverV` — 파형 저장 및 서빙

WAVE_RING의 파형을 디스크 **탱크(Tank) 파일**에 저장하고, 클라이언트 요청에 응답하는 와이어스피드 파형 서버입니다.

```ini
# waveserverV.d  –  대용량 설정 예시

MyModuleId       MOD_WAVESERVER
RingName         WAVE_RING
ServerPort       16022
GapThresh        1.0          # 갭 임계값(초)

# 쓰레드 설정 (대용량 핵심 파라미터)
NumWriters       8            # 디스크 쓰기 쓰레드 수
NumReaders       16           # 소켓 읽기 쓰레드 수
ClientTimeout    120          # 클라이언트 연결 타임아웃(초)

# 탱크 파일 정의
# 형식: Tank <SCNL> <탱크크기(MB)> <버퍼(초)> <경로>
Tank  SEO BH? KS 00  200  86400  /data/tanks/kma_bh.tnk
Tank  SEO HH? KS 00  500  43200  /data/tanks/kma_hh.tnk
Tank  *   BH? KS *   100  86400  /data/tanks/kma_all_bh.tnk

# 고성능 TCP 버퍼 설정
RcvBufLen        262144       # TCP 수신 버퍼 (256 KB)
SndBufLen        524288       # TCP 송신 버퍼 (512 KB)
```

## 5. 대용량 데이터 송신 (Export)

### 5.1 `export_generic` — TCP 파형 송신

WAVE_RING의 메시지를 TCP/IP를 통해 원격 시스템으로 전송합니다.

```ini
# export_generic.d  –  대용량 송신 설정

MyModuleId       MOD_EXPORT_GEN
RingName         WAVE_RING
HeartBeatInt     30

# 서버 모드 (수신 측이 연결해 오도록)
ServerSide       1
ServerPort       16005
MaxClients       20           # 최대 동시 클라이언트 수

# 송신할 메시지 타입
SendLogo  TYPE_TRACEBUF2   MOD_WILDCARD  INST_WILDCARD
SendLogo  TYPE_PICK_SCNL   MOD_WILDCARD  INST_WILDCARD
SendLogo  TYPE_HYPOINVERSE MOD_WILDCARD  INST_WILDCARD

# 성능 최적화 파라미터
MaxMsgSize       65536
QueueSize        8192         # 송신 큐 메시지 수 (대용량: 8192 이상)
SocketTimeout    30

# 압축 전송 (대역폭 절약)
Compress         1            # 1이면 zlib 압축 활성화
```

---

### 5.2 다중 목적지 동시 송신

하나의 Ring 데이터를 여러 외부 기관에 **동시 전송**하는 구성입니다.

```ini
# startstop.d  –  다중 export 모듈 등록 예시

# 국가지진센터 전송
Process     "export_generic export_to_kndc.d"

# IRIS DMC 전송
Process     "export_generic export_to_iris.d"

# 지역 모니터링 센터 전송
Process     "export_generic export_to_regional.d"

# 각 export 모듈은 독립된 .d 파일과 ModuleId 사용
# export_to_kndc.d:     MyModuleId MOD_EXPORT_KNDC,  ServerPort 16010
# export_to_iris.d:     MyModuleId MOD_EXPORT_IRIS,  ServerPort 16011
# export_to_regional.d: MyModuleId MOD_EXPORT_REG,   ServerPort 16012
```

---

### 5.3 `ew2slink` — SeedLink 형식 송신

TRACEBUF2 메시지를 **mini-SEED 레코드**로 변환하여 ringserver로 전송합니다. FDSN 호환 배포에 필수적입니다.

```ini
# ew2slink.d  –  SeedLink 송신 설정

MyModuleId       MOD_EW2SLINK
RingName         WAVE_RING

# SeedLink 서버(ringserver) 연결
MSeedHost        localhost
MSeedPort        16000

SeedRecordSize   512           # 512 또는 4096 바이트
GetLogo  TYPE_TRACEBUF2  MOD_WILDCARD  INST_WILDCARD

DefaultNetwork   KS            # SEED 네트워크 코드

# 성능 설정
QueueSize        2048
BatchSize        100           # 한 번에 ringserver에 보낼 레코드 수
```

## 6. 대용량 환경 성능 최적화

### 6.1 OS 커널 파라미터 튜닝

```bash
# /etc/sysctl.conf  –  Earthworm 대용량 최적화

# 공유 메모리 (Ring 크기보다 크게 설정)
kernel.shmmax = 2147483648      # 2 GB
kernel.shmall = 524288

# TCP 소켓 버퍼
net.core.rmem_max    = 134217728   # 수신 버퍼 최대 128 MB
net.core.wmem_max    = 134217728   # 송신 버퍼 최대 128 MB
net.ipv4.tcp_rmem    = 4096 87380 134217728
net.ipv4.tcp_wmem    = 4096 65536 134217728

# 연결 큐 및 파일 디스크립터
net.core.somaxconn              = 65535
net.ipv4.tcp_max_syn_backlog    = 65535
fs.file-max                     = 1048576

# 적용
sudo sysctl -p
```

```
# /etc/security/limits.conf  –  파일 수 제한 상향
earthworm  soft  nofile  65536
earthworm  hard  nofile  65536
```

### 6.2 WAVE_RING 크기 산정 공식

> **필요 Ring 크기 = 초당 데이터량(bytes/s) × 목표 보유 시간(초) × 안전 계수(3~5)**

#### 산정 예시

```
채널 수       : 500채널
샘플링 레이트 : 100 Hz 평균
데이터 타입   : int32 (4 bytes/sample)
초당 데이터량 : 500 × 100 × 4 = 200,000 bytes/s ≈ 200 KB/s
목표 보유 시간: 60초
안전 계수     : 4

필요 Ring 크기: 200 KB/s × 60s × 4 = 48 MB → 64 MB 이상 설정
```

| 채널 구성 | 예상 트래픽 | 권장 WAVE_RING 크기 |
|-----------|-------------|----------------------|
| 50채널 (BH 40Hz) | ~10 Mbps | 64 MB |
| 200채널 (BH+HH 혼합) | ~40 Mbps | 256 MB |
| 500채널 (HH 100Hz) | ~100 Mbps | 512 MB |
| 1000채널 (HH+EH 혼합) | ~250 Mbps | 1 GB |
| 2000채널 (광대역+강진동) | ~500 Mbps | 2 GB |

### 6.3 디스크 I/O 최적화

```bash
# /etc/fstab  –  XFS 마운트 옵션 최적화
/dev/sdb1  /data  xfs  defaults,noatime,nodiratime,allocsize=64m  0 0

# I/O 스케줄러 (SSD: none, HDD: mq-deadline)
echo mq-deadline > /sys/block/sdb/queue/scheduler

# 읽기 선행(Read-Ahead) 크기 증가
blockdev --setra 8192 /dev/sdb    # 4 MB read-ahead
```

> 💡 waveserverV 탱크 파일 전용 디스크는 **RAID-10(성능+가용성)** 구성을 권장합니다.

### 6.4 네트워크 대역폭 관리 (QoS)

```bash
# tc를 이용한 지진 데이터 트래픽 우선순위 설정
tc qdisc add dev eth0 root handle 1: htb default 30
tc class add dev eth0 parent 1: classid 1:1 htb rate 1000mbit

    match ip dport 16005 0xffff flowid 1:10
```

## 7. 고가용성(HA) 및 이중화 구성

### 7.1 이중화 패턴 비교

| 패턴 | 장점 | 단점 | 권장 용도 |
|------|------|------|-----------|
| **Active-Standby** | 단순, 안정적 | 대기 서버 유휴 | 소·중규모 네트워크 |
| **Active-Active** | 부하 분산 + 상호 백업 | 동기화 복잡 | 대규모 국가 네트워크 |
| **링 복제** | 실시간 데이터 미러링 | 네트워크 부하 증가 | 원격 백업 센터 |
| **탱크 공유(NFS)** | 두 서버가 동일 데이터 접근 | NFS 성능 병목 가능 | 읽기 중심 환경 |

### 9.3 Python(PyEW)으로 링 실시간 처리

```bash
pip install PyEW
```

```python
import PyEW
import struct, time

# Ring에 연결 (WAVE_RING=1000, 내 모듈 ID=200)
ring = PyEW.EWModule(ring_id=1000, mod_id=200, inst_id=141)
ring.add_ring(1000)

def parse_tracebuf2(data):
    """TRACEBUF2 헤더 파싱"""
    fmt = "ddd7s9s9s3s2s3shhi"
    hsize = struct.calcsize(fmt)
    hdr = struct.unpack(fmt, data[:hsize])
    return {
        "starttime": hdr[0],
        "endtime":   hdr[1],
        "samprate":  hdr[2],
        "sta":  hdr[3].decode().strip("\x00"),
        "net":  hdr[4].decode().strip("\x00"),
        "chan": hdr[5].decode().strip("\x00"),
        "nsamp": hdr[11],
    }

print("WAVE_RING 실시간 수신 시작...")
while True:
    msg = ring.get_wave(0)
    if msg:
        info = parse_tracebuf2(msg)
        delay = time.time() - info["starttime"]
        print(f"{info['net']}.{info['sta']}.{info['chan']} | "
              f"샘플수: {info['nsamp']} | 지연: {delay:.2f}초")
    time.sleep(0.001)
```

### 10.2 문제 해결 체크리스트

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| `Overflows > 0` | Ring 크기 부족 | `RingSize` 증가 |
| 모듈 재시작 반복 | 하트비트 미수신 | 로그 확인 및 설정 검토 |

| 탱크 파일 미생성 | 디스크 권한 문제 | `/data/tanks` 디렉토리 권한 확인 |
| SeedLink 연결 끊김 | 방화벽 또는 타임아웃 | `SocketTimeout`, `StateFile` 설정 |
| export 클라이언트 연결 실패 | `MaxClients` 초과 | `MaxClients` 값 증가 |
