## 1. 왜 NATS인가? — Earthworm과의 비교

### 1.1 Earthworm Transport Ring의 한계

Earthworm의 Transport Ring은 **단일 호스트 공유 메모리** 기반 IPC입니다. 수십 년간 검증된 안정성을 갖지만, 현대적인 분산 지진 관측 네트워크에서는 다음 한계가 드러납니다.

| 한계 | 내용 |
|------|------|
| 단일 호스트 | Ring은 물리적으로 한 서버에 종속 — 원격 배포 불가 |
| 고정 크기 버퍼 | Ring 오버플로우 시 오래된 메시지 유실 (복구 불가) |
| 프로토콜 폐쇄성 | Earthworm 전용 포맷 — 타 시스템 연동 복잡 |
| 수평 확장 불가 | 채널 수 증가 시 단순히 서버 사양만 높여야 함 |
| 메시지 재생 없음 | 유실된 데이터의 재요청·재전송 메커니즘 미존재 |

### 1.2 NATS가 제공하는 해법

```
Earthworm 구조                    NATS 구조
─────────────────                 ─────────────────────────────────
[모듈 A] ─▶ [공유메모리Ring] ◀─ [모듈 B]    [관측소A] ─▶ ┌───────────┐ ─▶ [분석서버]
              (단일호스트)                   [관측소B] ─▶ │ NATS      │ ─▶ [아카이브]
                                            [관측소C] ─▶ │ Cluster   │ ─▶ [경보시스템]
                                                         └───────────┘ ─▶ [웹대시보드]
                                                          (분산, 수평확장)
```

### 1.3 기능 비교표

| 기능 | Earthworm Ring | NATS Core | NATS JetStream |
|------|:--------------:|:---------:|:--------------:|
| 실시간 Pub/Sub | ✅ | ✅ | ✅ |
| 분산 네트워킹 | ❌ | ✅ | ✅ |
| 메시지 영속 저장 | ❌ | ❌ | ✅ |
| 메시지 재전송 | ❌ | ❌ | ✅ |
| 수평 확장(클러스터) | ❌ | ✅ | ✅ |
| 와일드카드 구독 | 제한적 | ✅ | ✅ |
| 전송 지연 (최저) | ~1 ms | **< 0.1 ms** | ~1 ms |
| 처리량 (최대) | ~수백 MB/s | **수십 GB/s** | ~수 GB/s |
| 다중 소비자 | 제한적 | ✅ (Fan-out) | ✅ (Consumer Group) |
| 메시지 ACK | ❌ | ❌ | ✅ |
| 클라우드/컨테이너 친화 | ❌ | ✅ | ✅ |

> 💡 **전략**: 초저지연 실시간 파형 배포는 **Core NATS**, 대용량 데이터 보장 전송·재처리는 **JetStream**을 함께 사용하는 하이브리드 구성이 최적입니다.

## 2. NATS 핵심 개념

### 2.1 메시지 모델

NATS의 메시지는 **Subject(토픽)** 에 발행(Publish)하고, 관심 있는 클라이언트가 구독(Subscribe)하는 **Pub/Sub** 모델입니다.

```
Publisher                NATS Server              Subscriber
─────────                ───────────              ──────────
[관측소: SEO]  ──pub──▶  seismic.KS.SEO.BHZ  ──▶ [분석서버]
                                               ──▶ [아카이브]
                                               ──▶ [경보시스템]
```

### 2.2 Subject 와일드카드

```
seismic.KS.SEO.BHZ        # 정확한 Subject
seismic.KS.SEO.*          # * : 단일 토큰 와일드카드  → SEO 모든 채널
seismic.KS.*.BHZ          # * : SEO, ICN, PUS 등 모든 KS 네트워크 BHZ
seismic.KS.>              # > : 다중 토큰 와일드카드  → KS 네트워크 전체
seismic.>                 # > : 모든 지진 데이터 전체
```

### 2.3 Core NATS vs JetStream

```
Core NATS (At-most-once)
  Publisher ──▶ NATS ──▶ Subscriber (현재 연결된 구독자에게만 전달)
  • 구독자 오프라인 시 메시지 유실
  • 초저지연, 초고처리량

JetStream (At-least-once / Exactly-once)
  Publisher ──▶ NATS ──▶ [Stream 저장소] ──▶ Consumer (재전송, ACK 보장)
  • 메시지 디스크 영속화
  • 오프라인 구독자도 나중에 수신 가능
  • 재처리·재생(Replay) 지원
```

### 2.4 JetStream 핵심 구성요소

| 구성요소 | 역할 | 지진 데이터 적용 |
|----------|------|-----------------|
| **Stream** | 메시지 저장소 (Subject 묶음) | `SEISMIC_WAVE` — 파형 전체 보관 |
| **Consumer** | Stream에서 메시지를 꺼내는 구독자 | 분석서버, 아카이브 서버 각각 독립 |
| **Subject Filter** | Consumer가 수신할 Subject 필터 | `seismic.KS.>` |
| **Retention** | 메시지 보존 정책 | `limits` (크기/시간) / `workqueue` |
| **Storage** | 저장 방식 | `file` (영속) / `memory` (임시) |
| **Ack Policy** | 수신 확인 방식 | `explicit` (안전) / `none` (고속) |

## 4. 지진 데이터 Subject 설계

NATS Subject는 Earthworm의 **SCNL(Station·Channel·Network·Location)** 체계에 자연스럽게 대응됩니다.

### 4.1 Subject 네이밍 컨벤션

```
seismic.<데이터타입>.<네트워크>.<관측소>.<채널>.<위치코드>

예시:
seismic.wave.KS.SEO.BHZ.00     ← KS 네트워크 서울 BHZ 채널 파형
seismic.wave.KS.SEO.BHN.00
seismic.wave.KS.ICN.HHZ.--
seismic.pick.KS.SEO.BHZ.00     ← P파 픽 결과
seismic.hypo.KS.>               ← 진앙 결정 결과
seismic.heartbeat.KS.SEO        ← 관측소 생존 신호
seismic.status.>                ← 시스템 상태

관리 채널:
_seismic.cmd.>                  ← 운영 명령 (언더스코어: 내부용)
```

### 4.2 Subject 구독 패턴 예시

```
# 특정 관측소 전 채널 수신
SUBSCRIBE seismic.wave.KS.SEO.>

# 전국 BHZ 채널만 수신 (수직성분)
SUBSCRIBE seismic.wave.*.*.BHZ.*

# KS 네트워크 전체 파형 + 픽 데이터
SUBSCRIBE seismic.wave.KS.>
SUBSCRIBE seismic.pick.KS.>

# 모든 지진 데이터 (모니터링 서버)
SUBSCRIBE seismic.>
```

## 5. Core NATS — 실시간 파형 스트리밍

Core NATS는 **< 0.1 ms 지연**으로 파형 데이터를 배포합니다. 구독자가 연결되어 있을 때만 전달되므로, 실시간 모니터링·경보 시스템에 최적입니다.

### 5.1 Python 송신기 (관측소 → NATS)

```python
# seismic_publisher.py
# pip install nats-py obspy numpy

import asyncio
import nats
import struct
import time
import json
import numpy as np
from dataclasses import dataclass, asdict

@dataclass
class TracePacket:
    """Earthworm TRACEBUF2 호환 파형 패킷"""
    network:    str
    station:    str
    channel:    str
    location:   str
    starttime:  float   # epoch seconds
    samprate:   float   # Hz
    nsamp:      int
    datatype:   str     # 'i4', 'f4'
    data:       list    # 샘플 배열

def pack_trace(packet: TracePacket) -> bytes:
    """TracePacket을 바이너리 직렬화 (헤더 + 데이터)"""
    header = struct.pack(
        '=ddi7s9s9s3s2s',
        packet.starttime,
        packet.starttime + packet.nsamp / packet.samprate,
        packet.samprate,
        packet.station.encode().ljust(7, b'\x00'),
        packet.network.encode().ljust(9, b'\x00'),
        packet.channel.encode().ljust(9, b'\x00'),
        packet.location.encode().ljust(3, b'\x00'),
        b'20',
    )
    samples = np.array(packet.data, dtype=np.int32).tobytes()
    return header + samples

async def seismic_publisher(nats_url: str = "nats://localhost:4222"):
    nc = await nats.connect(
        servers=[nats_url],
        name="seismic-publisher-SEO",
        reconnect_time_wait=2,
        max_reconnect_attempts=-1,      # 무한 재연결
        ping_interval=20,
        max_outstanding_pings=5,
    )
    print(f"✅ NATS 연결 완료: {nats_url}")

    # 관측소 설정
    stations = [
        {"net": "KS", "sta": "SEO", "cha": "BHZ", "loc": "00", "rate": 40.0},
        {"net": "KS", "sta": "SEO", "cha": "BHN", "loc": "00", "rate": 40.0},
        {"net": "KS", "sta": "SEO", "cha": "BHE", "loc": "00", "rate": 40.0},
    ]

    seq = 0
    try:
        while True:
            for st in stations:
                # 실제 환경에서는 digitizer에서 샘플 수집
                samples_per_packet = int(st["rate"])  # 1초치 데이터
                data = np.random.randint(-10000, 10000, samples_per_packet).tolist()

                packet = TracePacket(
                    network=st["net"],
                    station=st["sta"],
                    channel=st["cha"],
                    location=st["loc"],
                    starttime=time.time(),
                    samprate=st["rate"],
                    nsamp=samples_per_packet,
                    datatype="i4",
                    data=data,
                )

                # Subject: seismic.wave.KS.SEO.BHZ.00
                subject = f"seismic.wave.{st['net']}.{st['sta']}.{st['cha']}.{st['loc']}"
                payload = pack_trace(packet)

                await nc.publish(subject, payload)
                seq += 1

                if seq % 1000 == 0:
                    print(f"  📡 {seq}개 패킷 발행 완료 | 마지막: {subject}")

            await asyncio.sleep(1.0 / len(stations))  # 채널당 1초 간격

    except asyncio.CancelledError:
        await nc.drain()

asyncio.run(seismic_publisher())
```

### 5.2 Python 수신기 — 비동기 멀티채널 구독

```python
# seismic_subscriber.py

import asyncio
import nats
import struct
import numpy as np
from collections import defaultdict

def unpack_trace(data: bytes) -> dict:
    """바이너리 페이로드 파싱"""
    hdr_fmt = '=ddi7s9s9s3s2s'
    hdr_size = struct.calcsize(hdr_fmt)
    hdr = struct.unpack(hdr_fmt, data[:hdr_size])
    samples = np.frombuffer(data[hdr_size:], dtype=np.int32)
    return {
        "starttime": hdr[0],
        "endtime":   hdr[1],
        "samprate":  hdr[2],
        "station":   hdr[3].decode().strip('\x00'),
        "network":   hdr[4].decode().strip('\x00'),
        "channel":   hdr[5].decode().strip('\x00'),
        "location":  hdr[6].decode().strip('\x00'),
        "nsamp":     hdr[8] if len(hdr) > 8 else len(samples),
        "data":      samples,
    }

async def seismic_subscriber():
    nc = await nats.connect(
        servers=["nats://localhost:4222"],
        name="seismic-analyzer",
        reconnect_time_wait=2,
        max_reconnect_attempts=-1,
        error_cb=lambda e: print(f"❌ NATS 오류: {e}"),
        disconnected_cb=lambda: print("⚠️  연결 끊김 — 재연결 시도 중..."),
        reconnected_cb=lambda: print("✅ 재연결 완료"),
    )

    stats = defaultdict(int)

    async def wave_handler(msg):
        """파형 메시지 처리 핸들러"""
        try:
            trace = unpack_trace(msg.data)
            key = f"{trace['network']}.{trace['station']}.{trace['channel']}"
            stats[key] += 1

            delay = __import__('time').time() - trace['starttime']
            if delay > 2.0:
                print(f"⚠️  지연 감지 [{key}]: {delay:.2f}초")

            # 여기서 분석 로직 수행 (STA/LTA, 픽킹 등)
            await process_waveform(trace)

        except Exception as e:
            print(f"❌ 파싱 오류: {e}")

    async def process_waveform(trace: dict):
        """STA/LTA 기반 단순 이벤트 감지 예시"""
        data = trace['data'].astype(float)
        sta_len = int(trace['samprate'] * 0.5)   # 0.5초
        lta_len = int(trace['samprate'] * 10.0)  # 10초

        if len(data) < lta_len:
            return

        sta = np.mean(np.abs(data[-sta_len:]))
        lta = np.mean(np.abs(data[-lta_len:]))

        if lta > 0 and sta / lta > 3.0:
            key = f"{trace['network']}.{trace['station']}.{trace['channel']}"
            print(f"🚨 이벤트 감지! [{key}] STA/LTA = {sta/lta:.2f}")

    # 구독 등록
    sub_wave = await nc.subscribe("seismic.wave.KS.>", cb=wave_handler)
    print("✅ 구독 시작: seismic.wave.KS.>")

    # 통계 출력 (60초마다)
    try:
        while True:
            await asyncio.sleep(60)
            print(f"\n📊 수신 통계 (최근 60초):")
            for k, v in sorted(stats.items()):
                print(f"   {k}: {v} 패킷")
            stats.clear()
    except asyncio.CancelledError:
        await sub_wave.unsubscribe()
        await nc.drain()

asyncio.run(seismic_subscriber())
```

### 5.3 Request-Reply — 파형 데이터 요청

NATS의 Request-Reply 패턴을 이용해 특정 시간대 파형을 **직접 요청**할 수 있습니다.

```python
# 요청 측 (클라이언트)
async def request_waveform(nc, station, starttime, endtime):
    request = json.dumps({
```

### 6.4 JetStream Consumer 구성

```python
# jetstream_consumer.py

async def setup_consumers():
    nc = await nats.connect("nats://localhost:4222")
    js = nc.jetstream()

    # ── Consumer 1: 실시간 분석 서버 (Push, 큐 그룹) ──
    # 여러 분석 서버 인스턴스가 부하를 분산하여 처리
    async def analysis_handler(msg):
        try:
            trace = unpack_trace(msg.data)
            await run_analysis(trace)
            await msg.ack()           # 처리 완료 확인
        except Exception as e:
            print(f"❌ 분석 오류: {e}")
            await msg.nak(delay=5)    # 5초 후 재전송 요청

    await js.subscribe(
        "seismic.wave.KS.>",
        queue="analysis-workers",     # 큐 그룹: 여러 인스턴스 간 분산
        stream="SEISMIC_WAVE",
        durable="analysis-server",    # Consumer 이름 (재시작 후 이어받기)
        cb=analysis_handler,
        manual_ack=True,
        deliver_policy=nats.js.api.DeliverPolicy.NEW,  # 새 메시지부터
    )

    # ── Consumer 2: 아카이브 서버 (Pull, 배치 처리) ──
    # Pull Consumer는 스스로 속도를 제어하며 배치로 가져옴
    psub = await js.pull_subscribe(
        "seismic.wave.>",
        durable="archiver",
        stream="SEISMIC_WAVE",
    )

    while True:
        try:
            # 한 번에 최대 100개 메시지 가져오기
            msgs = await psub.fetch(batch=100, timeout=5.0)
            for msg in msgs:
                await archive_to_miniseed(msg.data)
                await msg.ack()
            print(f"  📦 아카이브: {len(msgs)}개 메시지 저장")
        except nats.errors.TimeoutError:
            pass  # 메시지 없으면 대기

asyncio.run(setup_consumers())
```

```python
async def batch_publisher(js, packets: list, batch_size=500):

    futures = []

            futures.clear()
```

## 11. Earthworm 연동 — 마이그레이션 전략

### 11.1 단계적 전환 아키텍처

전면 전환은 위험합니다. **Earthworm과 NATS를 병행 운영**하며 단계적으로 전환하는 방식을 권장합니다.

```
[ 1단계: 브릿지 연결 ]

  Earthworm             Bridge               NATS
  ─────────             ──────               ────
  WAVE_RING ──sniff──▶ ew2nats.py ──pub──▶ seismic.wave.>
                                            (JetStream)

[ 2단계: 신규 모듈 NATS 직접 연결 ]

  Earthworm                   NATS
  ─────────                   ────
  WAVE_RING ──▶ [레거시 모듈]  seismic.wave.> ──▶ [신규 분석서버]
                               seismic.pick.> ──▶ [신규 아카이브]

[ 3단계: Earthworm 단계적 제거 ]

  관측소 Digitizer ──▶ NATS만 사용 ──▶ 모든 하위 시스템
```

### 11.2 Earthworm → NATS 브릿지 구현

```python
# ew2nats_bridge.py
# Earthworm WAVE_RING 메시지를 NATS로 실시간 중계

import asyncio
import nats
import PyEW
import struct
import time

# 메시지 타입 매핑
EW_TYPE_TO_SUBJECT = {
    20: "seismic.wave",      # TYPE_TRACEBUF2
    10: "seismic.pick",      # TYPE_PICK_SCNL
    11: "seismic.pick",      # TYPE_CODA_SCNL
    12: "seismic.hypo",      # TYPE_HYP2000ARC
    101: "seismic.sm",       # TYPE_STRONGMOTION
}

async def ew2nats_bridge():
    # NATS 연결
    nc = await nats.connect(
        servers=["nats://localhost:4222"],
        name="ew2nats-bridge",
        max_reconnect_attempts=-1,
    )
    js = nc.jetstream()

    # Earthworm Ring 연결
    ring = PyEW.EWModule(ring_id=1000, mod_id=200, inst_id=141)
    ring.add_ring(1000)  # WAVE_RING

    bridge_count = 0
    error_count  = 0

    print("🌉 Earthworm → NATS 브릿지 시작")

    while True:
        try:
            # WAVE_RING에서 메시지 수신
            msg_type, msg_data = ring.get_msg()

            if msg_data is None:
                await asyncio.sleep(0.001)
                continue

            # TRACEBUF2 헤더에서 SCNL 파싱
            if msg_type == 20:  # TYPE_TRACEBUF2
                hdr = struct.unpack('=ddd7s9s9s3s2s', msg_data[:64])
                sta = hdr[3].decode().strip('\x00')
                net = hdr[4].decode().strip('\x00')
                cha = hdr[5].decode().strip('\x00')
                loc = hdr[6].decode().strip('\x00').replace(' ', '-')

                subject = f"seismic.wave.{net}.{sta}.{cha}.{loc}"

            elif msg_type in EW_TYPE_TO_SUBJECT:
                subject = f"{EW_TYPE_TO_SUBJECT[msg_type]}.bridge"
            else:
                continue

            # NATS JetStream으로 발행
            await js.publish(subject, msg_data, timeout=3.0)
            bridge_count += 1

            if bridge_count % 10000 == 0:
                print(f"  🌉 브릿지: {bridge_count}개 전달 | 오류: {error_count}개")

        except Exception as e:
            error_count += 1
            if error_count % 100 == 0:
                print(f"  ❌ 브릿지 오류 ({error_count}회): {e}")
            await asyncio.sleep(0.01)

asyncio.run(ew2nats_bridge())
```
