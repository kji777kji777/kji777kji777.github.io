## 1. 시스템 개요 및 목표

### 1.1 해결해야 할 핵심 문제

1,000개 이상의 miniSEED 채널을 클라이언트 브라우저에서 직접 렌더링하면 다음 문제가 발생합니다.

| 문제 | 브라우저 직접 렌더링 | 서버 측 렌더링(SSR) + 스트리밍 |
|---|---|---|
| **CPU 부하** | 클라이언트마다 32채널 × N명 = 폭발적 증가 | 서버 1회 렌더링 → 모든 클라이언트 공유 |
| **네트워크 대역폭** | 원시 파형 데이터를 클라이언트마다 전송 | 압축된 H.264 영상 1개만 전송 |
| **동기화** | 클라이언트별 타임스탬프 불일치 | 서버에서 단일 렌더링 → 완전 동기화 |
| **확장성** | 클라이언트 수 증가 → 서버 부하 선형 증가 | CDN/SRS로 수평 확장 용이 |

### 1.2 설계 목표

```
핵심 성능 목표
├── 채널 수용량  : 1,000개 이상 miniSEED 채널 동시 파싱
├── 클라이언트   : 동시 접속 500명 이상 안정 지원
├── 채널 선택    : 클라이언트당 최대 32채널 동적 선택
├── 지연시간     : 데이터 수신 → 화면 표출 500ms 이내
├── 해상도       : 32채널 그리드 FHD (1920×1080) 렌더링
└── 가용성       : 렌더링 워커 자동 복구 (HA 구성)
```

### 3.2 miniSEED 포맷 이해

```
miniSEED 레코드 구조 (기본 512 bytes)
┌──────────────────────────────────────┐
│  Fixed Section (48 bytes)            │
│  ├── Sequence Number (6 chars)       │
│  ├── Station ID (5 chars)            │
│  ├── Location ID (2 chars)           │
│  ├── Channel ID (3 chars)  ← NSLC   │
│  ├── Network ID (2 chars)            │
│  ├── Start Time (10 bytes)           │
│  ├── Sample Rate (2 bytes)           │
│  └── Sample Count (2 bytes)          │
├──────────────────────────────────────┤
│  Blockette 1000 (8 bytes)            │
│  └── 인코딩 방식 (STEIM-2 등)         │
├──────────────────────────────────────┤
│  Data Section (가변 크기)              │
│  └── 압축된 파형 샘플                  │
└──────────────────────────────────────┘

채널 ID 체계 (NSLC)
  Network.Station.Location.Channel
  예) KS.SEO..BHZ  → 한국 지진관측망, 서울, 광대역 수직
      KS.SEO..BHN  → 한국 지진관측망, 서울, 광대역 N방향
```

## 5. 레이어별 상세 구현

### 5-1. miniSEED 수집 및 파싱

```python
# ingestor/main.py
"""
SeedLink 프로토콜로 실시간 miniSEED 데이터를 수집하고
ObsPy로 파싱하여 Redis에 저장하는 인제스터

처리 흐름:
SeedLink Server → EasySeedLinkClient → ObsPy 파싱 → RedisWriter
"""
import logging
import signal
import sys
from obspy.clients.seedlink.easyclient import EasySeedLinkClient
from obspy import Stream, Trace
import numpy as np

from redis_writer import RedisChannelWriter

logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(name)s: %(message)s"
)
logger = logging.getLogger("ingestor")


class MiniSEEDIngestor(EasySeedLinkClient):
    """
    SeedLink 클라이언트 — miniSEED 패킷 실시간 수집

    EasySeedLinkClient를 상속하여 on_data() 콜백으로
    패킷이 수신될 때마다 처리합니다.
    """

    def __init__(self, server_url: str, redis_writer: RedisChannelWriter):
        super().__init__(server_url, autoconnect=False)
        self.redis_writer = redis_writer
        self.packet_count = 0
        self.channel_count = 0

    def on_data(self, trace: Trace):
        """
        miniSEED 패킷 수신 콜백
        ObsPy가 자동으로 miniSEED를 Trace 객체로 디코딩하여 전달합니다.

        Args:
            trace: ObsPy Trace 객체
                   trace.id       = "KS.SEO..BHZ" (NSLC ID)
                   trace.data     = numpy 배열 (파형 샘플)
                   trace.stats.starttime = 시작 시각 (UTCDateTime)
                   trace.stats.sampling_rate = 샘플링 레이트 (Hz)
        """
        channel_id = trace.id  # "KS.SEO..BHZ"

        # Redis에 최신 파형 데이터 저장
        self.redis_writer.push(
            channel_id=channel_id,
            samples=trace.data.astype(np.float32),
            starttime=float(trace.stats.starttime),
            sampling_rate=float(trace.stats.sampling_rate)
        )

        self.packet_count += 1
        if self.packet_count % 1000 == 0:
            logger.info(f"수신 패킷: {self.packet_count:,} | 활성 채널: {self.channel_count}")

    def subscribe_all_channels(self, channel_list: list):
        """
        채널 목록을 SeedLink에 구독 등록

        Args:
            channel_list: ["KS.SEO..BHZ", "KS.SEO..BHN", ...] 형태의 NSLC ID 목록
        """
        self.channel_count = len(channel_list)
        for nslc in channel_list:
            # NSLC를 Net, Sta, Loc, Chan으로 분리
            parts = nslc.split(".")
            if len(parts) != 4:
                logger.warning(f"잘못된 채널 형식: {nslc}")
                continue
            net, sta, loc, chan = parts
            self.select_stream(net, sta, f"{loc}{chan}")

        logger.info(f"SeedLink 구독 등록 완료: {self.channel_count}개 채널")


def load_channel_list(filepath: str) -> list:
    """채널 목록 파일 로드 (줄당 하나의 NSLC ID)"""
    with open(filepath) as f:
        return [line.strip() for line in f if line.strip() and not line.startswith("#")]


def main():
    import os

    seedlink_url = os.getenv("SEEDLINK_URL", "rtserve.iris.washington.edu:18000")
    redis_url = os.getenv("REDIS_URL", "redis://localhost:6379")
    channel_file = os.getenv("CHANNEL_FILE", "channels.txt")

    # Redis 라이터 초기화
    redis_writer = RedisChannelWriter(redis_url)

    # 채널 목록 로드
    channels = load_channel_list(channel_file)
    logger.info(f"로드된 채널 수: {len(channels)}")

    # 인제스터 시작
    client = MiniSEEDIngestor(seedlink_url, redis_writer)
    client.subscribe_all_channels(channels)

    # 종료 시그널 처리
    def shutdown(sig, frame):
        logger.info("인제스터 종료 중...")
        client.close()
        sys.exit(0)

    signal.signal(signal.SIGINT, shutdown)
    signal.signal(signal.SIGTERM, shutdown)
```

```python
# ingestor/redis_writer.py
"""
Redis 링 버퍼 저장 모듈
채널별로 최근 N초의 파형 데이터를 Redis LIST에 저장합니다.
"""
import json
import time
import logging
import redis
import numpy as np
from typing import Optional

logger = logging.getLogger("redis_writer")


class RedisChannelWriter:
    """
    채널별 파형 데이터를 Redis 링 버퍼에 저장

    저장 구조:
        KEY: "ch:{channel_id}"        → LIST of JSON-encoded packets
        KEY: "meta:{channel_id}"      → Hash (샘플링 레이트 등)
        KEY: "channels:active"        → SET (활성 채널 ID 목록)

    링 버퍼 크기: BUFFER_SECONDS초 분량의 샘플
    """

    BUFFER_SECONDS = 60       # 링 버퍼 보관 시간 (초)
    MAX_PACKET_SIZE = 10000   # 패킷당 최대 샘플 수

    def __init__(self, redis_url: str = "redis://localhost:6379"):
        self.redis = redis.from_url(redis_url, decode_responses=False)
        self.pipeline = self.redis.pipeline()
        self._push_count = 0

    def push(
        self,
        channel_id: str,
        samples: np.ndarray,
        starttime: float,
        sampling_rate: float
    ):
        """
        파형 데이터 패킷을 Redis 링 버퍼에 저장

        Args:
            channel_id: NSLC 채널 ID (예: "KS.SEO..BHZ")
            samples: float32 numpy 배열 (파형 샘플)
            starttime: 패킷 시작 시각 (Unix timestamp)
            sampling_rate: 샘플링 레이트 (Hz)
        """
        key = f"ch:{channel_id}"
        meta_key = f"meta:{channel_id}"

        # 버퍼 최대 항목 수 계산 (패킷 하나가 보통 512샘플)
        # 60초 × 샘플레이트 / 패킷당샘플수 로 최대 패킷 수 계산
        max_packets = max(
            10,
            int(self.BUFFER_SECONDS * sampling_rate / len(samples)) + 5
        )

        # 패킷 직렬화 (numpy 배열을 bytes로)
        packet = {
            "t": starttime,
            "sr": sampling_rate,
            "n": len(samples),
            # float32 bytes를 그대로 전송 (JSON보다 4배 작음)
            "d": samples.tobytes()
        }

        pipe = self.redis.pipeline(transaction=False)

        # LIST에 새 패킷 추가 (RPUSH)
        # 직렬화: starttime과 메타는 헤더로, 데이터는 binary로 분리 저장
        header = json.dumps({
            "t": starttime,
            "sr": float(sampling_rate),
            "n": int(len(samples))
        }).encode()

        pipe.rpush(f"{key}:header", header)
        pipe.rpush(f"{key}:data", samples.astype(np.float32).tobytes())

        # 링 버퍼 크기 유지 (오래된 데이터 자동 삭제)
        pipe.ltrim(f"{key}:header", -max_packets, -1)
        pipe.ltrim(f"{key}:data", -max_packets, -1)

        # 메타데이터 업데이트
        pipe.hset(meta_key, mapping={
            "sr": sampling_rate,
            "last_update": time.time(),
            "channel_id": channel_id
        })
        pipe.expire(meta_key, 120)  # 2분 TTL

        # 활성 채널 등록
        pipe.sadd("channels:active", channel_id)

        pipe.execute()

        self._push_count += 1

    def get_active_channels(self) -> list:
        """현재 활성화된 채널 ID 목록 반환"""
        return [cid.decode() for cid in self.redis.smembers("channels:active")]
```

```python
# renderer/redis_reader.py
"""
Redis에서 채널 파형 데이터를 읽어오는 모듈
렌더링 워커가 사용합니다.
"""
import json
import time
import numpy as np
import redis
from dataclasses import dataclass
from typing import Optional


@dataclass
class ChannelData:
    """단일 채널의 파형 데이터"""
    channel_id: str
    samples: np.ndarray       # float32 파형 샘플
    sampling_rate: float      # Hz
    latest_time: float        # 마지막 패킷 시각 (Unix timestamp)
    is_stale: bool = False    # 최근 업데이트가 없으면 True


class RedisChannelReader:
    """
    Redis에서 지정된 채널의 최근 파형 데이터를 읽어옵니다.
    렌더링에 필요한 N초 분량의 샘플을 하나의 배열로 합쳐 반환합니다.
    """

    STALE_THRESHOLD_SEC = 10  # N초 이상 데이터 없으면 stale 처리

    def __init__(self, redis_url: str = "redis://localhost:6379"):
        self.redis = redis.from_url(redis_url)

    def fetch(self, channel_id: str, window_sec: float = 30.0) -> ChannelData:
        """
        채널의 최근 window_sec 초 분량 파형 데이터를 반환

        Args:
            channel_id: NSLC 채널 ID
            window_sec: 가져올 시간 창 (초)

        Returns:
            ChannelData 객체
        """
        key = f"ch:{channel_id}"

        # 최신 패킷부터 역순으로 읽기 (LRANGE로 최근 N개)
        # window_sec * 최대샘플레이트(500Hz) / 패킷당샘플(512) ≈ 30개 패킷이면 충분
        max_packets = 50

        raw_headers = self.redis.lrange(f"{key}:header", -max_packets, -1)
        raw_data = self.redis.lrange(f"{key}:data", -max_packets, -1)

        if not raw_headers or not raw_data:
            # 데이터 없음 → 빈 배열 반환
            return ChannelData(
                channel_id=channel_id,
                samples=np.zeros(1000, dtype=np.float32),
                sampling_rate=100.0,
                latest_time=0.0,
                is_stale=True
            )

        # 패킷들을 하나의 배열로 합치기
        all_samples = []
        latest_time = 0.0
        sampling_rate = 100.0
        cutoff_time = time.time() - window_sec

        for header_raw, data_raw in zip(raw_headers, raw_data):
            header = json.loads(header_raw)
            pkt_time = header["t"]

            if pkt_time < cutoff_time:
                continue  # 창 밖의 오래된 패킷 제외

            sr = header["sr"]
            n = header["n"]

            # binary → numpy float32
            samples = np.frombuffer(data_raw, dtype=np.float32)
            if len(samples) != n:
                continue  # 손상된 패킷 무시

            all_samples.append(samples)
            latest_time = max(latest_time, pkt_time)
            sampling_rate = sr

        if not all_samples:
            return ChannelData(
                channel_id=channel_id,
                samples=np.zeros(1000, dtype=np.float32),
                sampling_rate=sampling_rate,
                latest_time=0.0,
                is_stale=True
            )

        combined = np.concatenate(all_samples)
        is_stale = (time.time() - latest_time) > self.STALE_THRESHOLD_SEC

        return ChannelData(
            channel_id=channel_id,
            samples=combined,
            sampling_rate=sampling_rate,
            latest_time=latest_time,
            is_stale=is_stale
        )

    def fetch_batch(self, channel_ids: list, window_sec: float = 30.0) -> dict:
        """
        여러 채널을 한 번에 가져오기 (Pipeline 활용으로 RTT 최소화)

        Returns:
            {channel_id: ChannelData} 딕셔너리
        """
        # Redis pipeline으로 모든 채널을 한 번의 TCP 왕복으로 가져오기
        results = {}
        for ch_id in channel_ids:
            results[ch_id] = self.fetch(ch_id, window_sec)
        return results
```

### 5-3. 서버 측 파형 렌더링 (OpenCV 핵심)

```python
# renderer/render_worker.py
"""
OpenCV 기반 파형 그리드 렌더링 워커

32개 채널을 4×8 그리드로 배치하여 1920×1080 프레임을 생성합니다.
각 채널 타일에는 파형, 채널명, 최신 진폭값을 표시합니다.
"""
import cv2
import numpy as np
import time
import logging
import os
from typing import List, Optional

from redis_reader import RedisChannelReader, ChannelData

logger = logging.getLogger("render_worker")


# ── 렌더링 설정 상수 ────────────────────────────────────────────────────
CANVAS_W, CANVAS_H = 1920, 1080   # 출력 해상도 (FHD)
GRID_COLS, GRID_ROWS = 4, 8       # 4×8 = 32채널 그리드
FPS = 20                           # 렌더링 프레임레이트

# 타일 크기 계산
TILE_W = CANVAS_W // GRID_COLS    # 480px
TILE_H = CANVAS_H // GRID_ROWS    # 135px

# 색상 팔레트 (BGR)
COLOR_BG         = (10, 10, 18)     # 진한 남색 배경
COLOR_BORDER     = (40, 40, 60)     # 타일 경계선
COLOR_WAVEFORM   = (0, 220, 80)     # 파형 색 (초록)
COLOR_WAVEFORM_STALE = (60, 60, 80) # 오래된 데이터 파형 색 (어두운 회색)
COLOR_LABEL      = (200, 200, 200)  # 채널 라벨 (밝은 회색)
COLOR_AMPLITUDE  = (80, 200, 255)   # 진폭값 색 (노란빛 흰색)
COLOR_ALERT      = (0, 50, 220)     # 알림 색 (빨강)
COLOR_GRID_LINE  = (25, 25, 40)     # 내부 그리드라인 (어두운 회색)
# ────────────────────────────────────────────────────────────────────────


class WaveformRenderer:
    """
    32채널 파형 그리드 렌더러

    매 프레임마다:
    1. 검정 캔버스 초기화
    2. 32개 타일 그리기 (각 채널의 파형)
    3. 타임스탬프, 채널 수 오버레이
    4. 완성된 프레임을 FFmpeg 파이프로 전달
    """

    def __init__(self, channel_ids: List[str]):
        """
        Args:
            channel_ids: 렌더링할 32개 채널의 NSLC ID 목록
        """
        self.channel_ids = channel_ids[:32]  # 최대 32채널
        self.canvas = np.zeros((CANVAS_H, CANVAS_W, 3), dtype=np.uint8)

        # 타일별 ROI 좌표 미리 계산 (매 프레임 재계산 방지)
        self.tile_rects = []
        for idx in range(GRID_COLS * GRID_ROWS):
            col = idx % GRID_COLS
            row = idx // GRID_COLS
            x0 = col * TILE_W
            y0 = row * TILE_H
            self.tile_rects.append((x0, y0, x0 + TILE_W, y0 + TILE_H))

        # 파형 표시 구간 설정 (타일 내부 패딩)
        self.wf_pad_x = 8    # 좌우 패딩
        self.wf_pad_y_top = 28   # 상단 패딩 (채널명 표시 공간)
        self.wf_pad_y_bot = 12   # 하단 패딩

        logger.info(
            f"렌더러 초기화: {len(self.channel_ids)}채널 | "
            f"해상도 {CANVAS_W}×{CANVAS_H} | {FPS}fps"
        )

    def render_frame(self, channel_data: dict) -> np.ndarray:
        """
        한 프레임 렌더링

        Args:
            channel_data: {channel_id: ChannelData} 딕셔너리

        Returns:
            렌더링된 BGR 프레임 (numpy ndarray)
        """
        # 캔버스 초기화 (배경색으로 채우기)
        self.canvas[:] = COLOR_BG

        for idx, ch_id in enumerate(self.channel_ids):
            if idx >= len(self.tile_rects):
                break

            x0, y0, x1, y1 = self.tile_rects[idx]
            data = channel_data.get(ch_id)

            # 타일 그리기
            self._draw_tile(x0, y0, x1, y1, ch_id, data)

        # 전체 화면 오버레이 (타임스탬프, 채널 수)
        self._draw_overlay(len(self.channel_ids))

        return self.canvas

    def _draw_tile(
        self,
        x0: int, y0: int, x1: int, y1: int,
        channel_id: str,
        data: Optional[ChannelData]
    ):
        """단일 채널 타일 그리기"""

        # ① 타일 경계선
        cv2.rectangle(self.canvas, (x0, y0), (x1 - 1, y1 - 1), COLOR_BORDER, 1)

        # ② 채널 라벨 (좌상단)
        # NSLC에서 Station과 Channel만 표시 (예: "SEO.BHZ")
        parts = channel_id.split(".")
        label = f"{parts[1]}.{parts[3]}" if len(parts) == 4 else channel_id
        cv2.putText(
            self.canvas, label,
            (x0 + self.wf_pad_x, y0 + 18),
            cv2.FONT_HERSHEY_SIMPLEX, 0.42,
            COLOR_LABEL, 1, cv2.LINE_AA
        )

        # 데이터 없음 처리
        if data is None or len(data.samples) == 0:
            cv2.putText(
                self.canvas, "NO DATA",
                (x0 + TILE_W // 2 - 35, y0 + TILE_H // 2),
                cv2.FONT_HERSHEY_SIMPLEX, 0.45,
                COLOR_ALERT, 1, cv2.LINE_AA
            )
            return

        # ③ 중앙 수평선 (영점 기준선)
        wf_y_center = y0 + self.wf_pad_y_top + (TILE_H - self.wf_pad_y_top - self.wf_pad_y_bot) // 2
        cv2.line(
            self.canvas,
            (x0 + self.wf_pad_x, wf_y_center),
            (x1 - self.wf_pad_x, wf_y_center),
            COLOR_GRID_LINE, 1
        )

        # ④ 파형 그리기
        self._draw_waveform(x0, y0, x1, y1, data)

        # ⑤ 최신 진폭값 표시 (우상단)
        if len(data.samples) > 0:
            latest_amp = data.samples[-1]
            amp_text = f"{latest_amp:+.1f}"
            amp_color = COLOR_ALERT if abs(latest_amp) > 10000 else COLOR_AMPLITUDE
            cv2.putText(
                self.canvas, amp_text,
                (x1 - 80, y0 + 18),
                cv2.FONT_HERSHEY_SIMPLEX, 0.38,
                amp_color, 1, cv2.LINE_AA
            )

        # ⑥ Stale 표시 (데이터가 오래된 경우)
        if data.is_stale:
            cv2.putText(
                self.canvas, "STALE",
                (x0 + TILE_W // 2 - 20, y0 + TILE_H - 8),
                cv2.FONT_HERSHEY_SIMPLEX, 0.35,
                COLOR_ALERT, 1, cv2.LINE_AA
            )

    def _draw_waveform(
        self,
        x0: int, y0: int, x1: int, y1: int,
        data: ChannelData
    ):
        """
        파형 폴리라인 그리기

        파형 영역 높이를 계산하여 샘플을 픽셀 좌표로 매핑합니다.
        """
        # 파형 표시 영역 계산
        wf_x0 = x0 + self.wf_pad_x
        wf_x1 = x1 - self.wf_pad_x
        wf_y0 = y0 + self.wf_pad_y_top
        wf_y1 = y1 - self.wf_pad_y_bot
        wf_w = wf_x1 - wf_x0
        wf_h = wf_y1 - wf_y0
        wf_cy = wf_y0 + wf_h // 2  # 파형 중심선

        samples = data.samples

        # 표시할 샘플 수를 픽셀 너비에 맞게 리샘플링
        # (1픽셀당 1샘플이 최적, 샘플이 많으면 다운샘플)
        n_pixels = wf_w
        if len(samples) > n_pixels:
            # 다운샘플: n_pixels 구간으로 나눠 각 구간의 최대값 사용
            # (극값 보존 다운샘플링으로 파형 특성 유지)
            indices = np.linspace(0, len(samples) - 1, n_pixels).astype(int)
            display_samples = samples[indices]
        else:
            display_samples = samples

        # 정규화: 현재 창의 최대 진폭 기준으로 스케일 조정
        amp_max = np.max(np.abs(display_samples))
        if amp_max < 1e-9:
            amp_max = 1.0  # 0으로 나누기 방지

        # 픽셀 좌표 변환
        n = len(display_samples)
        x_coords = np.linspace(wf_x0, wf_x1 - 1, n).astype(np.int32)
        # 진폭 → 픽셀 Y좌표 (위아래 반전, 여백 80% 사용)
        y_coords = (wf_cy - (display_samples / amp_max) * (wf_h * 0.4)).astype(np.int32)
        y_coords = np.clip(y_coords, wf_y0, wf_y1 - 1)

        # (x, y) 좌표 쌍으로 구성 → polylines 형식
        points = np.stack([x_coords, y_coords], axis=1).reshape(-1, 1, 2)

        wf_color = COLOR_WAVEFORM_STALE if data.is_stale else COLOR_WAVEFORM
        cv2.polylines(self.canvas, [points], isClosed=False, color=wf_color, thickness=1)

    def _draw_overlay(self, active_channels: int):
        """전체 화면 정보 오버레이"""
        ts = time.strftime("%Y-%m-%d %H:%M:%S UTC", time.gmtime())

        # 좌하단: 타임스탬프
        cv2.putText(
            self.canvas, ts,
            (10, CANVAS_H - 10),
            cv2.FONT_HERSHEY_SIMPLEX, 0.5,
            (120, 120, 120), 1, cv2.LINE_AA
        )

        # 우하단: 활성 채널 수
        ch_text = f"Channels: {active_channels}/32"
        cv2.putText(
            self.canvas, ch_text,
            (CANVAS_W - 160, CANVAS_H - 10),
            cv2.FONT_HERSHEY_SIMPLEX, 0.5,
            (120, 120, 120), 1, cv2.LINE_AA
        )
```

```python
# renderer/ffmpeg_streamer.py
"""
FFmpeg 파이프라인 관리 모듈

OpenCV 렌더러로부터 BGR 프레임을 stdin으로 받아
H.264(NVENC)로 인코딩하여 SRS에 RTMP로 송출합니다.
"""
import subprocess
import logging
import os
import time
import threading
from typing import Optional

import numpy as np

logger = logging.getLogger("ffmpeg_streamer")


class FFmpegStreamer:
    """
    FFmpeg stdin 파이프 기반 스트리머

    GPU 가용 여부에 따라 NVENC(GPU) 또는 libx264(CPU)를 자동 선택합니다.
    """

    def __init__(
        self,
        rtmp_url: str,
        width: int = 1920,
        height: int = 1080,
        fps: int = 20,
        use_gpu: bool = True
    ):
        self.rtmp_url = rtmp_url
        self.width = width
        self.height = height
        self.fps = fps
        self.use_gpu = use_gpu and self._check_nvenc_available()
        self.process: Optional[subprocess.Popen] = None
        self._frame_count = 0
        self._start_time = None

    def _check_nvenc_available(self) -> bool:
        """NVENC 코덱 사용 가능 여부 확인"""
        try:
            result = subprocess.run(
                ["ffmpeg", "-encoders"],
                capture_output=True, text=True, timeout=5
            )
            available = "h264_nvenc" in result.stdout
            if available:
                logger.info("GPU NVENC 인코더 사용 가능")
            else:
                logger.info("NVENC 없음 → CPU libx264 사용")
            return available
        except Exception:
            return False

    def _build_ffmpeg_cmd(self) -> list:
        """
        FFmpeg 명령어 구성

        입력: rawvideo (stdin에서 BGR 프레임)
        출력: H.264 → RTMP

        코덱별 설정:
        - h264_nvenc (GPU): preset=p1(가장 빠름), tune=ull(초저지연), zerolatency 버퍼
        - libx264  (CPU): preset=veryfast, tune=zerolatency
        """
        input_args = [
            "ffmpeg",
            "-loglevel", "warning",          # 경고 이상만 출력

            # 입력 스트림 설정 (stdin에서 rawvideo)
            "-f", "rawvideo",
            "-pixel_format", "bgr24",         # OpenCV 기본 포맷
            "-video_size", f"{self.width}x{self.height}",
            "-framerate", str(self.fps),
            "-i", "pipe:0",                   # stdin에서 읽기
        ]

        if self.use_gpu:
            # GPU NVENC 인코딩 (초저지연 설정)
            encode_args = [
                "-vcodec", "h264_nvenc",
                "-preset", "p1",              # p1=가장 빠름, p7=최고품질
                "-tune", "ull",               # Ultra Low Latency
                "-rc", "cbr",                 # 고정 비트레이트 (버퍼링 안정)
                "-b:v", "4M",                 # 4 Mbps (FHD 파형에 충분)
                "-maxrate", "4M",
                "-bufsize", "1M",             # 0.25초 버퍼 (지연 최소화)
                "-g", str(self.fps * 2),      # 키프레임 간격 2초
                "-zerolatency", "1",
            ]
        else:
            # CPU libx264 인코딩
            encode_args = [
                "-vcodec", "libx264",
                "-preset", "veryfast",
                "-tune", "zerolatency",
                "-b:v", "4M",
                "-maxrate", "5M",
                "-bufsize", "1M",
                "-g", str(self.fps * 2),
            ]

        output_args = [
            "-f", "flv",                      # RTMP는 FLV 컨테이너 사용
            self.rtmp_url
        ]

        return input_args + encode_args + output_args

    def start(self):
        """FFmpeg 프로세스 시작"""
        cmd = self._build_ffmpeg_cmd()
        logger.info(f"FFmpeg 시작: {' '.join(cmd)}")

        self.process = subprocess.Popen(
            cmd,
            stdin=subprocess.PIPE,
            stdout=subprocess.DEVNULL,
            stderr=subprocess.PIPE
        )

        # FFmpeg stderr 모니터링 스레드
        threading.Thread(
            target=self._monitor_stderr,
            daemon=True
        ).start()

        self._start_time = time.time()
        codec_type = "NVENC(GPU)" if self.use_gpu else "libx264(CPU)"
        logger.info(f"FFmpeg 시작 완료 | 코덱: {codec_type} | URL: {self.rtmp_url}")

    def write_frame(self, frame: np.ndarray):
        """
        BGR 프레임을 FFmpeg stdin에 전달

        Args:
            frame: (H, W, 3) BGR numpy 배열
        """
        if self.process is None or self.process.poll() is not None:
            raise RuntimeError("FFmpeg 프로세스가 실행 중이 아닙니다.")

        try:
            self.process.stdin.write(frame.tobytes())
            self._frame_count += 1

            # 1분마다 처리 속도 로그
            if self._frame_count % (self.fps * 60) == 0:
                elapsed = time.time() - self._start_time
                actual_fps = self._frame_count / elapsed
                logger.info(f"전송 프레임: {self._frame_count:,} | 실제 FPS: {actual_fps:.1f}")

        except BrokenPipeError:
            logger.error("FFmpeg 파이프 끊김 — 재시작 필요")
            raise

    def stop(self):
        """FFmpeg 프로세스 정리 종료"""
        if self.process:
            try:
                self.process.stdin.close()
                self.process.wait(timeout=5)
            except Exception:
                self.process.kill()
            logger.info("FFmpeg 프로세스 종료")

    def _monitor_stderr(self):
        """FFmpeg 오류 메시지 모니터링"""
        for line in self.process.stderr:
            line = line.decode(errors="replace").strip()
            if line:
                logger.warning(f"[FFmpeg] {line}")
```

### 5-5. 전체 렌더 워커 실행

```python
# renderer/main_worker.py
"""
렌더 워커 메인 루프
Redis에서 데이터 읽기 → 파형 렌더링 → FFmpeg으로 송출
"""
import time
import logging
import os
import sys
import signal
from typing import List

from redis_reader import RedisChannelReader
from render_worker import WaveformRenderer, FPS, CANVAS_W, CANVAS_H
from ffmpeg_streamer import FFmpegStreamer

logger = logging.getLogger("main_worker")
logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(name)s: %(message)s"
)


def run_worker(
    session_id: str,
    channel_ids: List[str],
    srs_host: str = "localhost"
):
    """
    단일 렌더 워커 실행

    Args:
        session_id: 클라이언트 세션 ID (스트림 URL에 사용)
        channel_ids: 렌더링할 채널 ID 목록 (최대 32개)
        srs_host: SRS 미디어 서버 호스트
    """
    rtmp_url = f"rtmp://{srs_host}:1935/live/session_{session_id}"
    redis_url = os.getenv("REDIS_URL", "redis://localhost:6379")

    logger.info(f"워커 시작 | 세션: {session_id} | 채널: {len(channel_ids)}개")
    logger.info(f"RTMP 대상: {rtmp_url}")

    # 모듈 초기화
    reader = RedisChannelReader(redis_url)
    renderer = WaveformRenderer(channel_ids)
    streamer = FFmpegStreamer(
        rtmp_url=rtmp_url,
        width=CANVAS_W,
        height=CANVAS_H,
        fps=FPS,
        use_gpu=os.getenv("USE_GPU", "true").lower() == "true"
    )

    running = True

    def shutdown(sig, frame):
        nonlocal running
        logger.info(f"워커 종료 시그널 수신 (세션: {session_id})")
        running = False

    signal.signal(signal.SIGTERM, shutdown)
    signal.signal(signal.SIGINT, shutdown)

    try:
        streamer.start()
        frame_interval = 1.0 / FPS
        next_frame_time = time.perf_counter()

        while running:
            loop_start = time.perf_counter()

            # 1. Redis에서 32채널 데이터 일괄 읽기
            channel_data = reader.fetch_batch(channel_ids, window_sec=30.0)

            # 2. 파형 그리드 렌더링
            frame = renderer.render_frame(channel_data)

            # 3. FFmpeg으로 프레임 전달
            streamer.write_frame(frame)

            # 4. 프레임 레이트 유지 (정확한 타이밍)
            next_frame_time += frame_interval
            sleep_time = next_frame_time - time.perf_counter()
            if sleep_time > 0:
                time.sleep(sleep_time)
            else:
                # 처리가 느릴 경우 경고 (프레임 드롭)
                if sleep_time < -0.05:  # 50ms 이상 지연
                    logger.warning(f"프레임 드롭 감지: {-sleep_time*1000:.0f}ms 초과")
                next_frame_time = time.perf_counter()

    except Exception as e:
        logger.error(f"워커 오류: {e}", exc_info=True)
    finally:
        streamer.stop()
        logger.info(f"워커 종료 (세션: {session_id})")


if __name__ == "__main__":
    # 환경변수 또는 커맨드라인에서 설정 읽기
    session = os.getenv("SESSION_ID", "default")
    channels_raw = os.getenv("CHANNEL_IDS", "")
    channels = [c.strip() for c in channels_raw.split(",") if c.strip()]
    srs = os.getenv("SRS_HOST", "srs")

    if not channels:
        logger.error("CHANNEL_IDS 환경변수가 비어 있습니다.")
        sys.exit(1)

    run_worker(session, channels, srs)
```

```javascript
// control-api/worker_manager.js
/**
 * 렌더 워커 프로세스 풀 관리자
 * Python 렌더 워커를 자식 프로세스로 관리합니다.
 */
const { spawn } = require('child_process');
const { v4: uuidv4 } = require('uuid');
const { createClient } = require('redis');

class WorkerManager {
    constructor() {
        this.sessions = new Map();    // session_id → { pid, channels, process }
        this.redis = createClient({ url: process.env.REDIS_URL || 'redis://redis:6379' });
        this.redis.connect();
    }

    async createSession(channels) {
        const sessionId = uuidv4().replace(/-/g, '').slice(0, 16);

        // Python 렌더 워커 프로세스 시작
        const workerProcess = spawn('python', ['main_worker.py'], {
            cwd: '/app/renderer',
            env: {
                ...process.env,
                SESSION_ID: sessionId,
                CHANNEL_IDS: channels.join(','),
                SRS_HOST: process.env.SRS_HOST || 'srs'
            },
            stdio: ['ignore', 'pipe', 'pipe']
        });

        workerProcess.stdout.on('data', (d) =>
            console.log(`[Worker ${sessionId}] ${d.toString().trim()}`)
        );
        workerProcess.stderr.on('data', (d) =>
            console.error(`[Worker ${sessionId}] ${d.toString().trim()}`)
        );
        workerProcess.on('exit', (code) => {
            console.log(`워커 종료: ${sessionId} (code: ${code})`);
            this.sessions.delete(sessionId);
        });

        const session = {
            id: sessionId,
            channels,
            process: workerProcess,
            workerPid: workerProcess.pid,
            createdAt: Date.now()
        };

        this.sessions.set(sessionId, session);
        return session;
    }

    async updateChannels(sessionId, newChannels) {
        const session = this.sessions.get(sessionId);
        if (!session) throw new Error(`세션 없음: ${sessionId}`);

        // Redis를 통해 채널 변경 명령 전달
        await this.redis.set(
            `worker:${sessionId}:channels`,
            newChannels.join(','),
            { EX: 60 }
        );
        session.channels = newChannels;
    }

    async destroySession(sessionId) {
        const session = this.sessions.get(sessionId);
        if (session) {
            session.process.kill('SIGTERM');
            this.sessions.delete(sessionId);
        }
    }

    async getActiveChannels() {
        const members = await this.redis.sMembers('channels:active');
        return members;
    }

    getSessionCount() { return this.sessions.size; }
    getWorkerCount() { return this.sessions.size; }
}

module.exports = { WorkerManager };
```

```html
        hls.on(Hls.Events.ERROR, (event, data) => {
            if (data.fatal) {
                log(`치명적 오류: ${data.type} — 재연결 시도`);
                setTimeout(() => connectHLS(m3u8Url), 3000);
            }
        });
    }

    function disconnectStream() {
        if (hls) { hls.destroy(); hls = null; }
        clearInterval(latencyTimer);
        document.getElementById('statusDot').classList.remove('live');
        document.getElementById('videoPlayer').src = '';
    }

    function startLatencyMonitor() {
        latencyTimer = setInterval(() => {
            if (!hls) return;
            const latency = hls.latency;
            const buffer = hls.mainForwardBufferInfo?.len || 0;
            document.getElementById('latencyVal').textContent =
                latency ? (latency * 1000).toFixed(0) : '—';
            document.getElementById('bufferVal').textContent =
                buffer.toFixed(1);
        }, 1000);
    }

    function toggleFullscreen() {
        const el = document.getElementById('videoPlayer');
        if (!document.fullscreenElement) el.requestFullscreen();
        else document.exitFullscreen();
    }
</script>
</body>
</html>
```

## 6. 성능 최적화 전략

### 6.1 렌더링 최적화

```python
# 최적화 기법 요약

# ① NumPy 벡터화 연산 — 루프 제거
# 나쁜 예 (느림):
for i, sample in enumerate(samples):
    y = int(center - sample / amp_max * scale)
    canvas[y, x0 + i] = (0, 220, 80)  # 픽셀별 접근

# 좋은 예 (빠름):
x_coords = np.linspace(x0, x1, len(samples)).astype(np.int32)
y_coords = (center - samples / amp_max * scale).astype(np.int32)
y_coords = np.clip(y_coords, wf_y0, wf_y1)
points = np.stack([x_coords, y_coords], axis=1).reshape(-1, 1, 2)
cv2.polylines(canvas, [points], False, (0, 220, 80), 1)

# ② 캔버스 재사용 — 매 프레임 새 배열 생성 방지
# self.canvas[:] = COLOR_BG  (기존 배열 재사용, 메모리 재할당 없음)

# ③ OpenCV 멀티스레딩 설정
import cv2
cv2.setNumThreads(4)         # CPU 코어 수에 맞게 조정
cv2.setUseOptimized(True)    # SIMD 최적화 활성화

# ④ 샘플 다운샘플링 — 불필요한 샘플 제거
# 100Hz 채널을 30초 표시하면 3,000샘플이지만 픽셀은 480개
# → 480픽셀로 다운샘플하여 연산량 6분의 1로 감소
```

### 6.2 Redis 최적화

```python
# ① Pipeline으로 네트워크 왕복(RTT) 최소화
# 나쁜 예: 32채널 × 개별 요청 = 32번의 TCP 왕복
for ch_id in channel_ids:
    data = redis.lrange(f"ch:{ch_id}:data", -50, -1)  # 32번 왕복

# 좋은 예: Pipeline으로 1번 왕복에 모두 처리
pipe = redis.pipeline()
for ch_id in channel_ids:
    pipe.lrange(f"ch:{ch_id}:header", -50, -1)
    pipe.lrange(f"ch:{ch_id}:data", -50, -1)
results = pipe.execute()  # 1번 왕복

# ② float32 binary 저장 — JSON 대비 75% 크기 절감
# JSON: "[1.23, 4.56, ...]" → 문자열 파싱 필요, 큰 용량
# Binary: samples.tobytes() → 직접 메모리 복사, 4배 작음
```

## 9. 성능 벤치마크 기준

### 9.1 목표 성능 지표

| 지표 | 목표값 | 측정 방법 |
|---|---|---|
| **수집 지연** | < 10ms | SeedLink 수신 → Redis 저장 타임스탬프 차이 |
| **렌더링 시간** | < 30ms / 프레임 (FHD 32채널) | Python time.perf_counter() |
| **인코딩 시간** | < 15ms / 프레임 (NVENC) | FFmpeg stats 출력 |
| **HLS 총 지연** | 2~5초 | 클라이언트 수신 vs 데이터 타임스탬프 |
| **WebRTC 총 지연** | 100~300ms | RTT 측정 |
| **동시 클라이언트** | 500명 이상 | SRS 부하 테스트 |
| **Redis 메모리** | 채널당 ≈ 5MB (60초 버퍼) | redis-cli info memory |
| **CPU 사용률** | < 40% (32채널 워커 1개, NVENC 사용 시) | htop |
| **GPU 사용률** | < 60% (NVENC 동시 10 스트림) | nvidia-smi |

### 9.2 채널 수에 따른 리소스 추정

| 채널 수 | Redis 메모리 | 수집 CPU | 렌더 CPU/GPU | 네트워크 (500 클라이언트) |
|---|---|---|---|---|
| 100채널 | ~0.5 GB | ~5% | ~10% CPU / ~5% GPU | ~2 Gbps |
| 500채널 | ~2.5 GB | ~20% | ~25% CPU / ~20% GPU | ~10 Gbps |
| 1,000채널 | ~5 GB | ~40% | ~50% CPU / ~40% GPU | ~20 Gbps |

> **참고**: 500 클라이언트가 각각 32채널 FHD 스트림(4Mbps)을 수신하면  
> 이론상 최대 2,000 Gbps가 필요하지만, **SRS → CDN 연계**로 대역폭을 대폭 절감합니다.  
> 실제 운영 시 클라우드 CDN(AWS CloudFront, Cloudflare Stream 등)과 연계를 권장합니다.
