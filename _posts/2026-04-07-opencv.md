## 1. 개요 및 배경

### 1.1 프로젝트 목적

기존 지진계(Seismometer)나 가속도 센서 없이, **일반 CCTV 또는 IP 카메라 영상**만으로 지진 발생을 실시간 감지하는 시스템을 개발합니다.

핵심 원리는 간단합니다.

```
지진 발생 → 건물/사물 진동 → 카메라에 포착 → 영상 분석 → 지진 여부 판별
```

영상 내 특정 사물(천장 조명, 식물, 캘린더 등)의 **움직임 벡터를 수치화**하여 지진의 주파수·진폭 특성과 비교함으로써 지진을 판별합니다.

### 1.2 동작 원리 개요

```
┌─────────────────────────────────────────────────────────┐
│  카메라 스트림 입력                                        │
│  (RTSP / USB / HLS)                                      │
└────────────────┬────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────┐
│  프레임 전처리 (OpenCV)                                    │
│  • 그레이스케일 변환                                        │
│  • 노이즈 필터링                                           │
│  • ROI (관심영역) 설정                                     │
└────────────────┬────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────┐
│  움직임 분석 엔진                                          │
│  • Optical Flow (Lucas-Kanade / Farneback)               │
│  • 움직임 벡터 크기·방향 계산                               │
│  • 주파수 분석 (FFT)                                       │
└────────────────┬────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────┐
│  지진 판별 알고리즘                                         │
│  • 움직임 패턴 특성 추출                                    │
│  • 임계값 기반 판별 / ML 모델 분류                          │
│  • 진도 추정                                               │
└────────────────┬────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────────┐
│  메시지 브로커 (NATS + Kafka)                              │
│  • 실시간 경보 발령 (NATS)                                  │
│  • 데이터 영속 저장 (Kafka)                                 │
└─────────────────────────────────────────────────────────┘
```

## 2. 기존 방식 vs 영상 판독 방식 비교

### 2.1 기존 지진 감지 방식

기존에는 주로 다음 세 가지 요소를 사용합니다.

- **지진계(Seismometer)**: 지면의 초미세 진동을 전기 신호로 변환
- **가속도 센서(Accelerometer)**: MEMS 기반 3축 가속도 측정
- **Earthworm 시스템**: 위 센서들의 데이터를 수집·처리하는 소프트웨어 플랫폼

### 2.2 방식별 상세 비교표

| 비교 항목 | 기존 센서 방식 | 영상 판독 방식 |
|---|---|---|
| **설치 비용** | 높음 (전문 센서: 수백만~수억 원) | 낮음 (기존 CCTV 활용 시 추가 비용 거의 없음) |
| **설치 난이도** | 높음 (전문가 설치, 기초 공사 필요) | 낮음 (카메라 위치 조정만 필요) |
| **감지 정밀도** | ⭐⭐⭐⭐⭐ (나노미터 단위 감지 가능) | ⭐⭐⭐ (픽셀 단위 제한, 소규모 진동 누락 가능) |
| **감지 한계 진도** | M0.1 이하도 감지 가능 | M2.0 이상에서 신뢰성 있게 감지 |
| **오경보율** | 매우 낮음 | 중간 (조명 변화, 사람 움직임 등 노이즈) |
| **설치 위치 제약** | 지반 직접 설치 필요 | 어디서나 가능 (건물 내부 포함) |
| **인프라 재활용** | 별도 인프라 필요 | 기존 CCTV 인프라 100% 재활용 |
| **대규모 배포** | 비용·시간 한계 | 쉬움 (소프트웨어 업데이트만으로 확장) |
| **부가 정보** | 진동 파형만 제공 | 영상 기반 상황 파악 동시 가능 |
| **유지보수** | 전문 장비 유지보수 필요 | 소프트웨어 업데이트 중심 |
| **야간/악천후** | 영향 없음 | 야간 카메라 또는 IR 카메라 필요 |
| **네트워크 의존성** | 낮음 | 높음 (영상 스트리밍 대역폭 필요) |

### 2.3 영상 방식의 장점 (상세)

**✅ 인프라 비용 절감**  
대한민국에는 이미 수백만 대의 CCTV가 설치되어 있습니다. 이 카메라들에 소프트웨어만 추가하면 전국적인 지진 감지 네트워크를 구축할 수 있습니다.

**✅ 밀도 높은 관측망**  
도심 지역은 센서보다 CCTV가 훨씬 많습니다. 더 촘촘한 관측 그물망으로 진앙 위치 추정 정확도가 향상됩니다.

**✅ 시각적 피해 정보 동시 제공**  
지진 감지와 동시에 건물 손상, 붕괴 여부 등 영상 기반 피해 정보를 제공할 수 있습니다.

**✅ 지속적인 알고리즘 개선**  
센서 교체 없이 소프트웨어 업데이트만으로 감지 알고리즘을 개선할 수 있습니다.

### 2.4 영상 방식의 단점 및 극복 방법

**⚠️ 소규모 지진 감지 한계**  
→ 극복: 여러 카메라의 데이터를 앙상블하여 감도 향상. 고감도 지진계와 보완적 사용.

**⚠️ 오경보 (False Positive)**  
→ 극복: 여러 카메라 동시 판별, ML 기반 노이즈 필터링, 지진 특유의 주파수 패턴 검증.

**⚠️ 야간 및 악천후**  
→ 극복: IR 카메라 활용, 야간 모드 카메라, 또는 실내 고정 카메라 우선 사용.

## 3. 시스템 아키텍처

### 3.1 전체 시스템 구성도

```
[카메라 레이어]
  CCTV / IP Camera (RTSP)
  USB 웹캠 (테스트용)
         │
         ▼
[영상 수집 레이어]
  FFmpeg 스트림 수신
  프레임 버퍼링
         │
         ▼
[분석 레이어] ◄─── 핵심 처리 영역
  ┌──────────────────────────────┐
  │  전처리 모듈                   │
  │  (노이즈 제거, ROI 설정)        │
  ├──────────────────────────────┤
  │  움직임 감지 모듈               │
  │  (Optical Flow, MOG2)         │
  ├──────────────────────────────┤
  │  지진 판별 모듈                 │
  │  (FFT 분석, ML 분류기)          │
  └──────────────────────────────┘
         │
         ▼
[메시지 레이어]
  NATS JetStream ──► 실시간 경보
  Apache Kafka   ──► 데이터 저장·재분석
         │
         ▼
[출력 레이어]
  경보 시스템 (SMS, Push, 사이렌)
  데이터 분석 대시보드
  Earthworm 연동 (선택)
```

### 3.2 데이터 흐름 및 처리 단계

| 단계 | 처리 내용 | 출력 데이터 | 처리 시간 목표 |
|---|---|---|---|
| **1. 프레임 수신** | 영상 스트림에서 프레임 추출 | Raw Frame (BGR) | < 33ms (30fps) |
| **2. 전처리** | 그레이스케일, 가우시안 블러 | Processed Frame | < 5ms |
| **3. Optical Flow** | 픽셀 움직임 벡터 계산 | Flow Vector Map | < 20ms |
| **4. 특성 추출** | 크기, 방향, 분산, 주파수 | Feature Vector | < 5ms |
| **5. 지진 판별** | 임계값 or ML 모델 추론 | 지진 여부 + 진도 | < 10ms |
| **6. 메시지 발행** | NATS/Kafka로 결과 전송 | JSON 메시지 | < 2ms |
| **총 파이프라인** | | | **< 65ms** |

## 4. 핵심 요소 기술

### 4.1 OpenCV (컴퓨터 비전 처리)

OpenCV(Open Source Computer Vision Library)는 이 프로젝트의 핵심 라이브러리입니다.

**사용하는 주요 기능:**

```
OpenCV 주요 기능
├── 영상 캡처
│   └── cv2.VideoCapture() — 카메라/스트림 연결
├── 전처리
│   ├── cv2.cvtColor() — 색상 공간 변환 (BGR → Gray)
│   └── cv2.GaussianBlur() — 노이즈 제거
├── 움직임 감지
│   ├── Optical Flow
│   │   ├── cv2.calcOpticalFlowPyrLK() — 희소 Optical Flow (빠름)
│   │   └── cv2.calcOpticalFlowFarneback() — 밀집 Optical Flow (정밀)
│   └── 배경 차분
│       └── cv2.createBackgroundSubtractorMOG2() — 움직임 마스크 추출
└── 분석
    └── numpy FFT — 주파수 도메인 분석
```

**Optical Flow란?**

Optical Flow는 연속된 두 프레임 사이에서 각 픽셀이 얼마나, 어느 방향으로 이동했는지를 계산합니다. 지진으로 건물이 흔들리면 영상 내 모든 정적 객체가 일제히 같은 방향으로 움직이는 패턴이 나타납니다.

```
프레임 t              프레임 t+1
┌────────────┐        ┌────────────┐
│  ●         │        │   ●        │  ← 조명이 오른쪽으로 이동
│            │   →    │            │
│  ■         │        │   ■        │  ← 물체도 같이 오른쪽으로 이동
└────────────┘        └────────────┘
   Optical Flow 벡터: → → → → → (일관된 방향 = 지진 의심)
```

### 4.2 NATS JetStream (실시간 경보)

NATS는 초저지연 메시지 전달에 특화된 브로커입니다. 지진 판별 즉시 경보를 전송할 때 사용합니다.

**역할**: 지진 감지 → 경보 발령 (1ms 이내 전달)

```
감지 노드 → NATS → 경보 시스템
              │
              └→ 모니터링 대시보드
              └→ 다른 분석 노드
```

### 4.3 Apache Kafka (데이터 파이프라인)

Kafka는 대용량 스트리밍 데이터의 영속적 저장과 재처리에 특화되어 있습니다.

**역할**: 원시 분석 데이터 수집 → 장기 보존 → ML 모델 학습용 데이터셋 구축

```
감지 노드 → Kafka Topic: raw-motion-data
              │
              ├→ Kafka Topic: earthquake-events
              ├→ ML 학습 파이프라인
              └→ 시계열 데이터베이스 (InfluxDB)
```

### 4.4 기술 스택 전체 요약

| 구분 | 기술 | 버전 | 역할 |
|---|---|---|---|
| **영상 처리** | OpenCV | 4.9+ | 프레임 분석, Optical Flow |
| **수치 연산** | NumPy | 1.26+ | 벡터 연산, FFT |

| **ML 모델** | scikit-learn / ONNX | - | 지진 패턴 분류 |
| **컨테이너** | Docker + Docker Compose | - | 환경 패키징 |
| **언어** | Python | 3.11+ | 메인 개발 언어 |

### 5.3 환경 설치 단계별 가이드

#### Step 1. 저장소 클론 및 가상환경 생성

```bash
# 프로젝트 클론
git clone https://github.com/your-org/vision-earthquake.git
cd vision-earthquake

# Python 가상환경 생성 (Python 3.11 권장)
python3.11 -m venv venv

# 가상환경 활성화
# Linux / macOS
source venv/bin/activate
# Windows
venv\Scripts\activate

# 프롬프트가 (venv)로 변경되면 성공
```

#### Step 2. Python 패키지 설치

```bash
# requirements.txt 내용:
# opencv-python==4.9.0.80
# numpy==1.26.4
# nats-py==2.6.0
# confluent-kafka==2.4.0
# scikit-learn==1.4.0
# scipy==1.13.0
# python-dotenv==1.0.0

pip install -r requirements.txt

# GPU 가속 사용 시 (CUDA 12.x 환경):
pip install opencv-python-headless  # 서버 환경용 (GUI 없음)
```

#### Step 3. Docker로 NATS + Kafka 실행

`docker-compose.yml` 파일을 생성합니다.

```yaml
# docker-compose.yml
version: '3.8'

services:
  # ── NATS JetStream (실시간 경보용) ──────────────────────────────
  nats:
    image: nats:2.10-alpine
```

### 6.1 영상 스트림 수신 모듈

```python
# src/capture/stream_reader.py
"""
영상 스트림 수신 모듈
- 웹캠, IP 카메라(RTSP), 영상 파일 모두 지원
- 프레임 드롭 감지 및 재연결 기능 포함
"""
import cv2
import time
import logging
from threading import Thread
from queue import Queue, Full

logger = logging.getLogger(__name__)


class StreamReader:
    """
    멀티스레드 기반 영상 스트림 리더
    
    별도 스레드에서 프레임을 수신하여 큐에 저장합니다.
    분석 스레드와 수신 스레드를 분리하여 프레임 드롭을 최소화합니다.
    """
    
    def __init__(self, source, buffer_size: int = 10):
        """
        Args:
            source: 카메라 인덱스(0,1...) 또는 RTSP URL 문자열
                    예) 0, "rtsp://192.168.1.100:554/stream"
            buffer_size: 프레임 버퍼 크기 (클수록 메모리 사용량 증가)
        """
        self.source = source
        self.cap = None
        self.frame_queue = Queue(maxsize=buffer_size)
        self.running = False
        self._thread = None
        
    def start(self):
        """스트림 수신 시작"""
        self.cap = cv2.VideoCapture(self.source)
        
        if not self.cap.isOpened():
            raise ConnectionError(f"카메라 연결 실패: {self.source}")
        
        # 카메라 속성 읽기
        self.fps = self.cap.get(cv2.CAP_PROP_FPS) or 30
        self.width = int(self.cap.get(cv2.CAP_PROP_FRAME_WIDTH))
        self.height = int(self.cap.get(cv2.CAP_PROP_FRAME_HEIGHT))
        
        logger.info(f"카메라 연결 성공: {self.width}x{self.height} @ {self.fps}fps")
        
        self.running = True
        self._thread = Thread(target=self._read_loop, daemon=True)
        self._thread.start()
        return self
    
    def _read_loop(self):
        """백그라운드 프레임 수신 루프"""
        reconnect_count = 0
        
        while self.running:
            ret, frame = self.cap.read()
            
            if not ret:
                # 프레임 수신 실패 시 재연결 시도
                reconnect_count += 1
                logger.warning(f"프레임 수신 실패 (시도 {reconnect_count}회)")
                
                if reconnect_count > 5:
                    logger.error("재연결 한계 초과. 스트림 종료.")
                    self.running = False
                    break
                
                time.sleep(1.0)
                self.cap.release()
                self.cap = cv2.VideoCapture(self.source)
                continue
            
            reconnect_count = 0
            timestamp = time.time()
            
            # 큐가 꽉 찼으면 가장 오래된 프레임 제거 후 새 프레임 추가
            try:
                self.frame_queue.put_nowait((timestamp, frame))
            except Full:
                try:
                    self.frame_queue.get_nowait()  # 오래된 프레임 제거
                    self.frame_queue.put_nowait((timestamp, frame))
                except Exception:
                    pass
    
    def read(self):
        """
        프레임 읽기
        
        Returns:
            tuple: (timestamp, frame) 또는 (None, None)
        """
        if not self.frame_queue.empty():
            return self.frame_queue.get()
        return None, None
    
    def stop(self):
        """스트림 수신 중지"""
        self.running = False
        if self.cap:
            self.cap.release()
        logger.info("스트림 수신 중지")
    
    def __enter__(self):
        return self.start()
    
    def __exit__(self, *args):
        self.stop()
```

```python
# src/analysis/preprocessor.py
"""
영상 전처리 모듈
- 노이즈 제거, 관심영역(ROI) 추출, 색상 변환 처리
"""
import cv2
import numpy as np
from dataclasses import dataclass
from typing import Optional, Tuple


@dataclass
class ROI:
    """관심영역 설정"""
    x: int
    y: int
    width: int
    height: int
    
    def apply(self, frame: np.ndarray) -> np.ndarray:
        """프레임에서 ROI 영역만 잘라냄"""
        return frame[self.y:self.y + self.height, self.x:self.x + self.width]


class Preprocessor:
    """
    영상 전처리기
    
    지진 감지에 유리하도록 영상을 전처리합니다.
    - 그레이스케일 변환: 색상 정보 제거로 연산량 감소
    - 가우시안 블러: 픽셀 노이즈 제거
    - 히스토그램 평활화: 조명 변화 보정
    """
    
    def __init__(
        self,
        roi: Optional[ROI] = None,
        blur_kernel: Tuple[int, int] = (5, 5),
        equalize_histogram: bool = True
    ):
        """
        Args:
            roi: 관심영역 (None이면 전체 프레임 사용)
            blur_kernel: 가우시안 블러 커널 크기 (홀수여야 함)
            equalize_histogram: 히스토그램 평활화 적용 여부
        """
        self.roi = roi
        self.blur_kernel = blur_kernel
        self.equalize_histogram = equalize_histogram
    
    def process(self, frame: np.ndarray) -> np.ndarray:
        """
        프레임 전처리 수행
        
        Args:
            frame: BGR 형식 원본 프레임
            
        Returns:
            전처리된 그레이스케일 프레임
        """
        # 1. ROI 적용
        if self.roi:
            frame = self.roi.apply(frame)
        
        # 2. 그레이스케일 변환 (BGR → Gray)
        gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
        
        # 3. 가우시안 블러 (고주파 노이즈 제거)
        blurred = cv2.GaussianBlur(gray, self.blur_kernel, 0)
        
        # 4. 히스토그램 평활화 (조명 변화 보정)
        if self.equalize_histogram:
            blurred = cv2.equalizeHist(blurred)
        
        return blurred
    
    def draw_roi(self, frame: np.ndarray) -> np.ndarray:
        """원본 프레임에 ROI 영역을 시각화"""
        if self.roi:
            cv2.rectangle(
                frame,
                (self.roi.x, self.roi.y),
                (self.roi.x + self.roi.width, self.roi.y + self.roi.height),
                (0, 255, 0), 2  # 초록색 사각형
            )
        return frame
```

### 6.3 움직임 감지 모듈 (핵심)

```python
# src/analysis/motion_detector.py
"""
움직임 감지 모듈 — 이 프로젝트의 핵심 알고리즘
Optical Flow를 사용하여 프레임 간 픽셀 움직임을 계산합니다.
"""
import cv2
import numpy as np
from dataclasses import dataclass, field
from typing import Optional, List
from collections import deque


@dataclass
class MotionData:
    """단일 프레임의 움직임 분석 결과"""
    timestamp: float            # 프레임 타임스탬프
    mean_magnitude: float       # 평균 움직임 크기 (픽셀)
    max_magnitude: float        # 최대 움직임 크기
    mean_direction: float       # 평균 움직임 방향 (라디안)
    direction_variance: float   # 방향의 분산 (작을수록 일관된 방향 = 지진 의심)
    coverage_ratio: float       # 움직임이 감지된 영역 비율 (0~1)
    flow_vectors: Optional[np.ndarray] = field(default=None, repr=False)


class OpticalFlowDetector:
    """
    Farneback Dense Optical Flow 기반 움직임 감지기
    
    Dense Optical Flow는 모든 픽셀의 움직임을 계산합니다.
    지진 시 나타나는 특성:
    1. 대부분의 픽셀이 같은 방향으로 이동 (direction_variance ↓)
    2. 움직임 크기가 주기적으로 변화 (지진파 주파수)
    3. 화면 전체에서 움직임 발생 (coverage_ratio ↑)
    """
    
    def __init__(
        self,
        min_magnitude: float = 0.5,    # 노이즈 제거를 위한 최소 움직임 크기
        history_size: int = 90,         # 분석에 사용할 히스토리 프레임 수 (3초 @ 30fps)
    ):
        self.min_magnitude = min_magnitude
        self.history: deque = deque(maxlen=history_size)
        self.prev_frame: Optional[np.ndarray] = None
        
        # Farneback Optical Flow 파라미터
        # (학술 논문 기반 최적화된 기본값)
        self.flow_params = dict(
            pyr_scale=0.5,    # 피라미드 스케일 (0.5 = 절반씩 축소)
            levels=3,          # 피라미드 레벨 수
            winsize=15,        # 윈도우 크기 (클수록 부드럽지만 느림)
            iterations=3,      # 각 레벨에서의 반복 횟수
            poly_n=5,          # 픽셀 근방 크기
            poly_sigma=1.2,    # 가우시안 표준편차
            flags=0
        )
    
    def detect(self, timestamp: float, frame: np.ndarray) -> Optional[MotionData]:
        """
        움직임 감지 수행
        
        Args:
            timestamp: 현재 프레임의 타임스탬프
            frame: 전처리된 그레이스케일 프레임
            
        Returns:
            MotionData (첫 프레임에서는 None 반환)
        """
        # 첫 프레임은 이전 프레임이 없으므로 저장만 하고 반환
        if self.prev_frame is None:
            self.prev_frame = frame
            return None
        
        # Dense Optical Flow 계산
        # flow 배열: (H, W, 2) — 각 픽셀의 (dx, dy) 움직임 벡터
        flow = cv2.calcOpticalFlowFarneback(
            self.prev_frame,
            frame,
            None,
            **self.flow_params
        )
        
        # 움직임 크기와 방향 계산
        # magnitude: 각 픽셀이 얼마나 이동했는지 (픽셀 단위)
        # angle: 어느 방향으로 이동했는지 (라디안)
        magnitude, angle = cv2.cartToPolar(flow[..., 0], flow[..., 1])
        
        # 노이즈 필터링: 최소 크기 이하는 움직임으로 보지 않음
        motion_mask = magnitude > self.min_magnitude
        
        if not np.any(motion_mask):
            # 움직임 없음
            motion_data = MotionData(
                timestamp=timestamp,
                mean_magnitude=0.0,
                max_magnitude=0.0,
                mean_direction=0.0,
                direction_variance=0.0,
                coverage_ratio=0.0
            )
        else:
            valid_magnitudes = magnitude[motion_mask]
            valid_angles = angle[motion_mask]
            
            motion_data = MotionData(
                timestamp=timestamp,
                mean_magnitude=float(np.mean(valid_magnitudes)),
                max_magnitude=float(np.max(valid_magnitudes)),
                mean_direction=float(np.mean(valid_angles)),
                # 방향 분산이 낮으면 모든 픽셀이 같은 방향 → 지진 가능성 ↑
                direction_variance=float(np.var(valid_angles)),
                # 전체 픽셀 중 움직임이 감지된 비율
                coverage_ratio=float(np.sum(motion_mask)) / motion_mask.size,
                flow_vectors=flow
            )
        
        # 히스토리에 추가
        self.history.append(motion_data)
        
        # 현재 프레임을 다음 프레임의 이전 프레임으로 저장
        self.prev_frame = frame
        
        return motion_data
    
    def get_magnitude_time_series(self) -> np.ndarray:
        """
        히스토리에서 움직임 크기의 시계열 데이터 추출
        FFT 분석에 사용됩니다.
        """
        return np.array([m.mean_magnitude for m in self.history])
    
    def visualize_flow(self, frame: np.ndarray, flow: np.ndarray) -> np.ndarray:
        """
        Optical Flow를 HSV 색상으로 시각화
        
        색상 → 움직임 방향
        채도 → 움직임 크기
        """
        magnitude, angle = cv2.cartToPolar(flow[..., 0], flow[..., 1])
        
        hsv = np.zeros((*frame.shape, 3), dtype=np.uint8)
        hsv[..., 0] = angle * 180 / np.pi / 2  # 색조 (방향)
        hsv[..., 1] = 255                        # 채도
        hsv[..., 2] = cv2.normalize(magnitude, None, 0, 255, cv2.NORM_MINMAX)
        
        return cv2.cvtColor(hsv, cv2.COLOR_HSV2BGR)
```

```python
# src/analysis/earthquake_classifier.py
"""
지진 판별 모듈
움직임 데이터에서 지진 특성을 추출하고 판별합니다.
"""
import numpy as np
from scipy import signal
from scipy.fft import rfft, rfftfreq
from dataclasses import dataclass
from typing import List
from collections import deque

from .motion_detector import MotionData


@dataclass
class EarthquakeResult:
    """지진 판별 결과"""
    timestamp: float
    is_earthquake: bool         # 지진 여부
    confidence: float           # 신뢰도 (0.0 ~ 1.0)
    estimated_intensity: str    # 추정 진도 ("I" ~ "VII+")
    dominant_frequency: float   # 지배 주파수 (Hz)
    features: dict              # 판별에 사용된 특성값들


class EarthquakeClassifier:
    """
    규칙 기반 지진 판별기
    
    지진 파형의 물리적 특성을 이용한 판별 규칙:
    1. 주파수 범위: 지진은 주로 0.1 ~ 10 Hz 대역에서 발생
    2. 움직임 방향 일관성: 지진 시 화면 내 사물이 같은 방향으로 흔들림
    3. 움직임 커버리지: 지진 시 화면 전체가 영향을 받음
    4. 진동 지속시간: 일시적 충격(문 닫힘 등)과 구별
    """
    
    # 지진 주파수 범위 (Hz)
    # 건물의 고유진동수: 저층 2~5Hz, 고층 0.2~1Hz
    EARTHQUAKE_FREQ_MIN = 0.1
    EARTHQUAKE_FREQ_MAX = 10.0
    
    # 진도 추정 기준 (평균 픽셀 움직임 기준, 카메라 환경에 따라 조정 필요)
    INTENSITY_THRESHOLDS = {
        "I":    (0.5,  2.0),    # 미진: 거의 느끼지 못함
        "II":   (2.0,  5.0),    # 경미: 정지 상태의 사람만 느낌
        "III":  (5.0,  10.0),   # 약진: 건물 내 사람이 느낌
        "IV":   (10.0, 20.0),   # 중진: 대부분의 사람이 느낌
        "V":    (20.0, 40.0),   # 강진: 모든 사람이 느낌
        "VI+":  (40.0, float('inf'))  # 격진 이상
    }
    
    def __init__(
        self,
        fps: float = 30.0,
        window_size: int = 90,          # 분석 창 크기 (3초 @ 30fps)
        confidence_threshold: float = 0.65
    ):
        self.fps = fps
        self.window_size = window_size
        self.confidence_threshold = confidence_threshold
    
    def classify(self, motion_history: List[MotionData]) -> EarthquakeResult:
        """
        움직임 히스토리를 분석하여 지진 여부 판별
        
        Args:
            motion_history: 최근 N 프레임의 움직임 데이터 리스트
            
        Returns:
            EarthquakeResult
        """
        if len(motion_history) < self.window_size // 3:
            # 데이터 부족
            return EarthquakeResult(
                timestamp=motion_history[-1].timestamp if motion_history else 0,
                is_earthquake=False,
                confidence=0.0,
                estimated_intensity="N/A",
                dominant_frequency=0.0,
                features={"error": "insufficient_data"}
            )
        
        # 특성 추출
        features = self._extract_features(motion_history)
        
        # 규칙 기반 점수 계산 (각 규칙이 0~1점)
        scores = []
        
        # 규칙 1: 지진 주파수 범위 내에 에너지가 있는가?
        freq_score = self._score_frequency(features['dominant_freq'])
        scores.append(('frequency', freq_score, 0.35))  # 가중치 35%
        
        # 규칙 2: 움직임 방향이 일관적인가? (분산이 낮은가?)
        direction_score = self._score_direction_consistency(features['direction_variance'])
        scores.append(('direction', direction_score, 0.25))
        
        # 규칙 3: 움직임이 화면 전체에 걸쳐 있는가?
        coverage_score = self._score_coverage(features['mean_coverage'])
        scores.append(('coverage', coverage_score, 0.20))
        
        # 규칙 4: 움직임 크기가 임계값 이상인가?
        magnitude_score = self._score_magnitude(features['mean_magnitude'])
        scores.append(('magnitude', magnitude_score, 0.20))
        
        # 가중 평균으로 최종 신뢰도 계산
        confidence = sum(score * weight for _, score, weight in scores)
        
        is_earthquake = confidence >= self.confidence_threshold
        
        return EarthquakeResult(
            timestamp=motion_history[-1].timestamp,
            is_earthquake=is_earthquake,
            confidence=round(confidence, 4),
            estimated_intensity=self._estimate_intensity(features['mean_magnitude']) if is_earthquake else "N/A",
            dominant_frequency=round(features['dominant_freq'], 2),
            features={
                'mean_magnitude': round(features['mean_magnitude'], 3),
                'direction_variance': round(features['direction_variance'], 3),
                'mean_coverage': round(features['mean_coverage'], 3),
                'dominant_freq': round(features['dominant_freq'], 3),
                'scores': {name: round(s, 3) for name, s, _ in scores}
            }
        )
    
    def _extract_features(self, history: List[MotionData]) -> dict:
        """움직임 히스토리에서 지진 판별용 특성 추출"""
        magnitudes = np.array([m.mean_magnitude for m in history])
        directions = np.array([m.mean_direction for m in history])
        coverages = np.array([m.coverage_ratio for m in history])
        
        # FFT로 주파수 분석
        # 어떤 주파수의 진동이 가장 강한지 찾기
        fft_vals = np.abs(rfft(magnitudes))
        freqs = rfftfreq(len(magnitudes), d=1.0 / self.fps)
        
        # 가장 에너지가 강한 주파수 찾기
        if len(fft_vals) > 1:
            dominant_idx = np.argmax(fft_vals[1:]) + 1  # DC 성분(0Hz) 제외
            dominant_freq = freqs[dominant_idx]
        else:
            dominant_freq = 0.0
        
        return {
            'mean_magnitude': float(np.mean(magnitudes)),
            'max_magnitude': float(np.max(magnitudes)),
            'direction_variance': float(np.mean(np.var(directions.reshape(-1, 1), axis=1))),
            'mean_coverage': float(np.mean(coverages)),
            'dominant_freq': float(dominant_freq),
        }
    
    def _score_frequency(self, dominant_freq: float) -> float:
        """지진 주파수 범위에 있으면 높은 점수"""
        if self.EARTHQUAKE_FREQ_MIN <= dominant_freq <= self.EARTHQUAKE_FREQ_MAX:
            # 2~5Hz 범위에서 최고점 (건물 고유진동수)
            if 2.0 <= dominant_freq <= 5.0:
                return 1.0
            return 0.7
        return 0.1
    
    def _score_direction_consistency(self, direction_variance: float) -> float:
        """방향이 일관적일수록(분산이 낮을수록) 높은 점수"""
        # 분산이 0.5 이하이면 매우 일관됨
        if direction_variance < 0.5:
            return 1.0
        elif direction_variance < 1.5:
            return 0.7
        elif direction_variance < 3.0:
            return 0.4
        return 0.1
    
    def _score_coverage(self, mean_coverage: float) -> float:
        """화면의 넓은 영역에서 움직임이 감지될수록 높은 점수"""
        if mean_coverage > 0.7:
            return 1.0
        elif mean_coverage > 0.4:
            return 0.6
        elif mean_coverage > 0.2:
            return 0.3
        return 0.0
    
    def _score_magnitude(self, mean_magnitude: float) -> float:
        """최소 움직임 크기 이상일 때 점수 부여"""
        if mean_magnitude > 5.0:
            return 1.0
        elif mean_magnitude > 2.5:
            return 0.7
        elif mean_magnitude > 1.0:
            return 0.4
        return 0.0
    
    def _estimate_intensity(self, mean_magnitude: float) -> str:
        """움직임 크기로 대략적인 진도 추정"""
        for intensity, (low, high) in self.INTENSITY_THRESHOLDS.items():
            if low <= mean_magnitude < high:
                return intensity
        return "VI+"
```

```python
# src/messaging/nats_publisher.py
"""
NATS JetStream 발행 모듈
지진 감지 즉시 경보를 초저지연으로 전달합니다.
"""
import asyncio
import json
import logging
from datetime import datetime, timezone

import nats
from nats.js.api import StreamConfig

logger = logging.getLogger(__name__)


class NATSPublisher:
    """
    NATS JetStream 기반 이벤트 발행자
    
    발행하는 Subject:
    - earthquake.alert  : 지진 감지 경보 (즉시 발행)
    - earthquake.raw    : 원시 움직임 데이터 (매 프레임)
    """
    
    def __init__(self, nats_url: str = "nats://localhost:4222"):
        self.nats_url = nats_url
        self.nc = None   # NATS 연결 객체
        self.js = None   # JetStream 컨텍스트
    
    async def connect(self):
        """NATS 서버에 연결 및 JetStream 스트림 생성"""
        self.nc = await nats.connect(self.nats_url)
        self.js = self.nc.jetstream()
        
        # earthquake-events 스트림 생성 (없으면 자동 생성)
        try:
            await self.js.add_stream(
                StreamConfig(
                    name="earthquake-events",
                    subjects=["earthquake.>"],  # earthquake.* 모든 주제 포함
                    max_age=86400,  # 24시간 보관
                    max_bytes=1_000_000_000,  # 최대 1GB
                )
            )
            logger.info("JetStream 스트림 생성 완료: earthquake-events")
        except Exception as e:
            if "stream name already in use" in str(e):
                logger.info("JetStream 스트림 이미 존재: earthquake-events")
            else:
                raise
    
    async def publish_alert(self, result) -> bool:
        """
        지진 경보 발행
        
        Args:
            result: EarthquakeResult 객체
            
        Returns:
            발행 성공 여부
        """
        message = {
            "event_type": "earthquake_detected",
            "timestamp": datetime.fromtimestamp(
                result.timestamp, tz=timezone.utc
            ).isoformat(),
            "is_earthquake": result.is_earthquake,
            "confidence": result.confidence,
            "estimated_intensity": result.estimated_intensity,
            "dominant_frequency_hz": result.dominant_frequency,
            "features": result.features,
        }
        
        try:
            ack = await self.js.publish(
                "earthquake.alert",
                json.dumps(message).encode()
            )
            logger.info(
                f"경보 발행 완료 | seq={ack.seq} | "
                f"신뢰도={result.confidence:.2%} | "
                f"진도={result.estimated_intensity}"
            )
            return True
        except Exception as e:
            logger.error(f"경보 발행 실패: {e}")
            return False
    
    async def publish_raw(self, motion_data) -> bool:
        """원시 움직임 데이터 발행 (매 프레임, 경량 메시지)"""
        message = {
            "timestamp": motion_data.timestamp,
            "mean_magnitude": motion_data.mean_magnitude,
            "coverage_ratio": motion_data.coverage_ratio,
            "direction_variance": motion_data.direction_variance,
        }
        
        try:
            await self.js.publish(
                "earthquake.raw",
                json.dumps(message).encode()
            )
            return True
        except Exception as e:
            logger.debug(f"원시 데이터 발행 실패: {e}")
            return False
    
    async def close(self):
        """연결 종료"""
        if self.nc:
            await self.nc.drain()
            await self.nc.close()
```

### 7.2 Kafka 프로듀서 모듈 (데이터 영속화)

```python
# src/messaging/kafka_producer.py
"""
Kafka 프로듀서 모듈
움직임 데이터를 Kafka에 저장하여 장기 보존 및 재분석을 지원합니다.
"""
import json
import logging
from datetime import datetime, timezone
from typing import Optional

from confluent_kafka import Producer
from confluent_kafka.admin import AdminClient, NewTopic

logger = logging.getLogger(__name__)


class KafkaProducerWrapper:
    """
    Kafka 프로듀서 래퍼
    
    토픽:
    - motion-data      : 모든 프레임의 움직임 데이터 (고빈도)
    - earthquake-events: 지진 감지 이벤트만 (저빈도)
    """
    
    TOPICS = {
        "motion-data": {
            "num_partitions": 3,
            "replication_factor": 1,
            "config": {
                "retention.ms": str(7 * 24 * 60 * 60 * 1000),  # 7일 보관
                "compression.type": "lz4",  # 압축으로 저장 공간 절약
            }
        },
        "earthquake-events": {
            "num_partitions": 1,
            "replication_factor": 1,
            "config": {
                "retention.ms": str(365 * 24 * 60 * 60 * 1000),  # 1년 보관
            }
        }
    }
    
    def __init__(self, bootstrap_servers: str = "localhost:9092"):
        self.bootstrap_servers = bootstrap_servers
        self.producer: Optional[Producer] = None
    
    def connect(self):
        """Kafka 연결 및 토픽 초기화"""
        conf = {
            "bootstrap.servers": self.bootstrap_servers,
            "client.id": "earthquake-detector",
            # 성능 최적화 설정
            "batch.size": 65536,        # 64KB 배치 전송
            "linger.ms": 10,            # 10ms 대기 후 배치 전송
            "compression.type": "lz4",  # 메시지 압축
        }
        
        self.producer = Producer(conf)
        self._create_topics()
        logger.info(f"Kafka 연결 완료: {self.bootstrap_servers}")
    
    def _create_topics(self):
        """필요한 Kafka 토픽 생성"""
        admin = AdminClient({"bootstrap.servers": self.bootstrap_servers})
        
        new_topics = [
            NewTopic(
                topic_name,
                num_partitions=config["num_partitions"],
                replication_factor=config["replication_factor"],
                config=config["config"]
            )
            for topic_name, config in self.TOPICS.items()
        ]
        
        results = admin.create_topics(new_topics)
        for topic, future in results.items():
            try:
                future.result()
                logger.info(f"Kafka 토픽 생성: {topic}")
            except Exception as e:
                if "already exists" in str(e).lower():
                    logger.debug(f"Kafka 토픽 이미 존재: {topic}")
                else:
                    logger.error(f"토픽 생성 실패 {topic}: {e}")
    
    def send_motion_data(self, motion_data):
        """프레임별 움직임 데이터 전송"""
        message = {
            "ts": motion_data.timestamp,
            "mag": round(motion_data.mean_magnitude, 4),
            "max_mag": round(motion_data.max_magnitude, 4),
            "dir": round(motion_data.mean_direction, 4),
            "dir_var": round(motion_data.direction_variance, 4),
            "cov": round(motion_data.coverage_ratio, 4),
        }
        
        self.producer.produce(
            topic="motion-data",
            key=str(int(motion_data.timestamp * 1000)).encode(),
            value=json.dumps(message).encode(),
            callback=self._delivery_callback
        )
        # 비동기 전송 (poll로 콜백 처리)
        self.producer.poll(0)
    
    def send_earthquake_event(self, result):
        """지진 이벤트 전송 (동기 방식 — 데이터 손실 방지)"""
        message = {
            "timestamp": datetime.fromtimestamp(
                result.timestamp, tz=timezone.utc
            ).isoformat(),
            "is_earthquake": result.is_earthquake,
            "confidence": result.confidence,
            "intensity": result.estimated_intensity,
            "dominant_freq_hz": result.dominant_frequency,
            "features": result.features,
        }
        
        self.producer.produce(
            topic="earthquake-events",
            value=json.dumps(message).encode(),
        )
        # 지진 이벤트는 즉시 플러시 (데이터 손실 방지)
        self.producer.flush(timeout=5.0)
        logger.info(f"Kafka 지진 이벤트 전송 완료: {message['timestamp']}")
    
    def _delivery_callback(self, err, msg):
        """메시지 전송 결과 콜백"""
        if err:
            logger.error(f"Kafka 전송 실패: {err}")
    
    def close(self):
        """프로듀서 종료"""
        if self.producer:
            self.producer.flush(timeout=10.0)
            logger.info("Kafka 프로듀서 종료")
```

## 8. 전체 파이프라인 통합

```python
# src/main.py
"""
지진 감지 시스템 메인 진입점
모든 모듈을 통합하여 실시간 파이프라인을 구성합니다.
"""
import asyncio
import logging
import os
import signal
import time
from dotenv import load_dotenv

from capture.stream_reader import StreamReader
from analysis.preprocessor import Preprocessor, ROI
from analysis.motion_detector import OpticalFlowDetector
from analysis.earthquake_classifier import EarthquakeClassifier
from messaging.nats_publisher import NATSPublisher
from messaging.kafka_producer import KafkaProducerWrapper

# 로깅 설정
logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(name)s: %(message)s",
    datefmt="%Y-%m-%d %H:%M:%S"
)
logger = logging.getLogger("main")

# 환경 변수 로드
load_dotenv()


class EarthquakeDetectionPipeline:
    """
    전체 지진 감지 파이프라인을 조율하는 메인 클래스
    
    처리 흐름:
    StreamReader → Preprocessor → OpticalFlowDetector 
    → EarthquakeClassifier → NATSPublisher / KafkaProducer
    """
    
    def __init__(self):
        # 설정 로드
        self.camera_source = os.getenv("CAMERA_SOURCE", "0")
        try:
            self.camera_source = int(self.camera_source)  # 숫자이면 웹캠 인덱스
        except ValueError:
            pass  # 문자열이면 RTSP URL
        
        self.fps = float(os.getenv("CAMERA_FPS", "30"))
        
        # 모듈 초기화
        self.stream = StreamReader(self.camera_source)
        
        roi_config = ROI(
            x=int(os.getenv("ROI_X", "0")),
            y=int(os.getenv("ROI_Y", "0")),
            width=int(os.getenv("ROI_W", "640")),
            height=int(os.getenv("ROI_H", "480"))
        )
        
        self.preprocessor = Preprocessor(roi=roi_config)
        self.motion_detector = OpticalFlowDetector(
            min_magnitude=float(os.getenv("MOTION_THRESHOLD", "0.5"))
        )
        self.classifier = EarthquakeClassifier(
            fps=self.fps,
            confidence_threshold=float(os.getenv("EARTHQUAKE_THRESHOLD", "0.65"))
        )
        
        # 메시지 브로커
        self.nats_pub = NATSPublisher(os.getenv("NATS_URL", "nats://localhost:4222"))
        self.kafka_prod = KafkaProducerWrapper(
            os.getenv("KAFKA_BOOTSTRAP_SERVERS", "localhost:9092")
        )
        
        self.running = False
        self.frame_count = 0
        self.earthquake_count = 0
        
        # 분석 주기 설정 (매 N 프레임마다 지진 판별 수행)
        self.classify_every_n_frames = int(self.fps)  # 1초마다 판별
    
    async def run(self):
        """파이프라인 실행"""
        logger.info("=" * 60)
        logger.info("  영상 기반 지진 감지 시스템 시작")
        logger.info("=" * 60)
        
        # 연결 초기화
        await self.nats_pub.connect()
        self.kafka_prod.connect()
        self.stream.start()
        
        self.running = True
        logger.info(f"카메라 소스: {self.camera_source}")
        logger.info("실시간 모니터링 시작...")
        
        try:
            await self._main_loop()
        finally:
            await self._cleanup()
    
    async def _main_loop(self):
        """메인 처리 루프"""
        last_classify_time = time.time()
        
        while self.running:
            timestamp, frame = self.stream.read()
            
            if frame is None:
                await asyncio.sleep(0.001)
                continue
            
            self.frame_count += 1
            
            # 1. 전처리
            processed = self.preprocessor.process(frame)
            
            # 2. 움직임 감지
            motion_data = self.motion_detector.detect(timestamp, processed)
            
            if motion_data is None:
                continue
            
            # 3. Kafka에 원시 데이터 전송 (비동기, 고빈도)
            self.kafka_prod.send_motion_data(motion_data)
            
            # 4. 지진 판별 (매 N 프레임마다)
            if self.frame_count % self.classify_every_n_frames == 0:
                history = list(self.motion_detector.history)
                result = self.classifier.classify(history)
                
                # 지진 감지 시 즉시 경보
                if result.is_earthquake:
                    self.earthquake_count += 1
                    logger.warning(
                        f"⚠️  지진 감지! | 신뢰도={result.confidence:.1%} | "
                        f"추정 진도={result.estimated_intensity} | "
                        f"주파수={result.dominant_frequency}Hz"
                    )
                    
                    # NATS로 즉각 경보 발행
                    await self.nats_pub.publish_alert(result)
                    
                    # Kafka에 이벤트 저장
                    self.kafka_prod.send_earthquake_event(result)
                
                # 상태 로그 (매 30초마다)
                current_time = time.time()
                if current_time - last_classify_time > 30:
                    logger.info(
                        f"상태: 처리 프레임={self.frame_count} | "
                        f"감지 횟수={self.earthquake_count} | "
                        f"현재 움직임={motion_data.mean_magnitude:.2f}px"
                    )
                    last_classify_time = current_time
            
            # CPU 과부하 방지
            await asyncio.sleep(0)
    
    async def _cleanup(self):
        """종료 시 리소스 정리"""
        logger.info("시스템 종료 중...")
        self.stream.stop()
        self.kafka_prod.close()
        await self.nats_pub.close()
        logger.info(
            f"종료 완료 | 총 처리 프레임: {self.frame_count} | "
            f"감지된 지진: {self.earthquake_count}회"
        )


async def main():
    pipeline = EarthquakeDetectionPipeline()
    
    # Ctrl+C 처리
    loop = asyncio.get_event_loop()
    for sig in (signal.SIGINT, signal.SIGTERM):
        loop.add_signal_handler(sig, lambda: setattr(pipeline, 'running', False))
    
    await pipeline.run()


if __name__ == "__main__":
    asyncio.run(main())
```

```python
# tests/test_motion_detector.py
"""
움직임 감지 모듈 단위 테스트
실제 카메라 없이 합성 영상으로 테스트
"""
import numpy as np
import pytest
import time

from src.analysis.motion_detector import OpticalFlowDetector
from src.analysis.earthquake_classifier import EarthquakeClassifier


def generate_static_frame(width=320, height=240) -> np.ndarray:
    """정적 배경 프레임 생성"""
    frame = np.zeros((height, width), dtype=np.uint8)
    # 체커보드 패턴 (움직임 감지 용이)
    frame[::20, :] = 128
    frame[:, ::20] = 128
    return frame


def generate_shaking_frame(base_frame, dx=5, dy=3) -> np.ndarray:
    """지진처럼 흔들린 프레임 생성 (이동 변환)"""
    import cv2
    M = np.float32([[1, 0, dx], [0, 1, dy]])
    return cv2.warpAffine(base_frame, M, (base_frame.shape[1], base_frame.shape[0]))


class TestOpticalFlowDetector:
    
    def setup_method(self):
        self.detector = OpticalFlowDetector(min_magnitude=0.3)
        self.base_frame = generate_static_frame()
    
    def test_first_frame_returns_none(self):
        """첫 프레임은 이전 프레임이 없으므로 None 반환해야 함"""
        result = self.detector.detect(time.time(), self.base_frame)
        assert result is None
    
    def test_static_scene_low_magnitude(self):
        """정적 장면에서 움직임이 거의 없어야 함"""
        self.detector.detect(time.time(), self.base_frame)
        result = self.detector.detect(time.time() + 0.033, self.base_frame)
        
        assert result is not None
        assert result.mean_magnitude < 1.0, "정적 장면에서 큰 움직임이 감지됨"
    
    def test_shaking_scene_high_magnitude(self):
        """흔들린 장면에서 큰 움직임이 감지되어야 함"""
        self.detector.detect(time.time(), self.base_frame)
        
        shaken = generate_shaking_frame(self.base_frame, dx=10, dy=8)
        result = self.detector.detect(time.time() + 0.033, shaken)
        
        assert result is not None
        assert result.mean_magnitude > 2.0, "흔들린 장면에서 움직임이 감지되지 않음"
    
    def test_consistent_direction_on_earthquake(self):
        """지진 시뮬레이션: 모든 픽셀이 같은 방향으로 이동"""
        self.detector.detect(time.time(), self.base_frame)
        
        # 일정한 방향으로 이동 (지진 특성)
        shaken = generate_shaking_frame(self.base_frame, dx=8, dy=0)
        result = self.detector.detect(time.time() + 0.033, shaken)
        
        assert result is not None
        # 방향 분산이 낮아야 함 (일관된 방향)
        assert result.direction_variance < 2.0, "지진 시뮬레이션에서 방향이 불일치"

    def test_earthquake_pattern_detected(self):
        """지진 패턴에서 감지되어야 함"""
        from src.analysis.motion_detector import MotionData
        import math
        
        # 2Hz 지진파 시뮬레이션 (30프레임 = 1초)
        history = []
        for i in range(60):
            t = i / 30.0
            # 2Hz 사인파로 움직임 크기 시뮬레이션
            magnitude = 10.0 + 8.0 * math.sin(2 * math.pi * 2 * t)
            history.append(MotionData(
                timestamp=time.time() + t,
                mean_magnitude=magnitude,
                max_magnitude=magnitude * 1.5,
                mean_direction=0.1,           # 일관된 방향 (거의 0)
                direction_variance=0.2,        # 낮은 분산 (일관성)
                coverage_ratio=0.75            # 화면 75% 커버리지
            ))
        
        result = self.classifier.classify(history)
        # 지진 패턴이므로 감지되거나 높은 신뢰도를 가져야 함
        assert result.confidence > 0.5, f"지진 패턴 신뢰도가 낮음: {result.confidence}"
```
